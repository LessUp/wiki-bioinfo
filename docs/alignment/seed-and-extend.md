# Seed-and-extend

## 是什么

Seed-and-extend 是现代序列搜索与 read mapping 中非常核心的策略：

1. 先快速找到一段可靠的局部匹配（seed）；
2. 再围绕这些候选位置做更精细的扩展（extend）。

它把“全局搜索一个近似匹配”拆成了“先缩小候选范围，再局部精细求解”。

## 要解决什么生物信息学问题

直接在整个参考序列上对每条 read 做完整近似匹配，代价通常太高。

但如果我们能先找到一个足够可信的小片段，就可以：

- 快速锁定候选位置；
- 把昂贵的动态规划只放在少量局部区域；
- 同时保留对 mismatch / indel 的容忍能力。

## 输入与输出

- 输入：query 序列、参考索引、seeding 规则、扩展评分模型
- 输出：候选比对位置及其局部最优对齐结果

## 核心思想 / 数学模型

一个典型流程是：

- 使用 k-mer、minimizer、FM-index 等方法寻找精确 seed；
- 根据多个 seed 的位置关系过滤候选；
- 对保留下来的位置运行局部动态规划或其他扩展策略；
- 输出最终得分、坐标和可能的 CIGAR。

## worked example

假设一条 read 长 100bp，其中只有 80bp 与参考高度一致，中间还有少量错配和一个短 indel。

如果要求全长一开始就精确匹配，这条 read 可能根本找不到位置；
但如果先在其中找到一个稳定 seed，再做局部扩展，就仍然有机会恢复正确比对。

## 复杂度与适用前提

Seed-and-extend 的优势在于：

- 搜索快；
- 扩展只在少数候选区域发生；
- 能兼顾真实数据中的误差和变异。

但它也依赖 seed 设计：

- seed 太短，假阳性多；
- seed 太长，容易错过真实匹配；
- 低复杂度或高重复区域会让 seeding 变得困难。

## 与真实工具或流程的连接

从 BLAST 到很多 read mapper，这种“先找 seed、再扩展”的思想都非常常见。它是把索引结构和动态规划结合起来的典型工程路线。

## 常见误区

### 只要有 seed 就一定能恢复正确比对

不一定。seed 的质量、重复区域、错误分布和扩展模型都会影响最终结果。

### extend 只是补一下边角，不重要

不是。真正决定 gap、错配和局部最优路径的往往就是扩展阶段。
