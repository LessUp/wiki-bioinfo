# 组装评估

## 是什么

组装评估关注的是：得到 contig 或 scaffold 之后，怎样判断结果是否足够完整、连续、准确，以及这些指标是否真的符合你的分析目标。

## 要解决什么生物信息学问题

一个组装结果看起来“很长”并不一定就更好。我们真正关心的是：

- 连续性如何；
- 是否有明显错组装；
- 是否覆盖了目标区域；
- 对下游注释和比较分析是否足够可靠。

## 核心指标

### 连续性

常见代表指标包括 N50 等，它们描述组装片段长度分布，但不能单独说明正确性。

### 完整性

完整性关心的是：目标基因组或转录组中应有的内容，是否被足够覆盖。

### 正确性

错误连接、重复塌陷、局部错序等问题，都可能让组装在连续性看起来不错时仍然不可用。

## 为什么不能只看一个数字

N50 很容易被误用。它能反映片段是否够长，但不能保证：

- 路径是否正确；
- 重复区域是否被错误合并；
- 生物学上重要的区域是否被完整保留。

## 与真实工具或流程的连接

真实组装流程中，评估往往需要结合：

- 长度统计；
- 与参考或保守基因集的比较；
- 下游注释是否稳定；
- 特定研究问题的目标区域是否被恢复。

## 常见误区

### N50 越大，组装一定越好

不对。更大的 N50 可能来自错误拼接。

### 只要能出 contig，就说明组装已经成功

也不对。组装结果是否可用于下游分析，取决于它是否真实、完整、稳定。
