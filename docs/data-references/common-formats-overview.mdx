## 是什么

在基因组或转录组分析中，数据通常不会直接从“原始实验”跳到“最终结论”。它会在多个层级上逐步流动，每一层都对应不同的文件格式：

- **序列层**：FASTA；
- **原始观测层**：FASTQ；
- **定位结果层**：SAM/BAM/CRAM；
- **注释与区间层**：GTF/GFF/BED；
- **变异结果层**：VCF/BCF；
- **派生指标层**：表达矩阵、coverage、统计汇总等。

这页的重点不是逐个拆字段，而是先回答：**一个格式处在哪一层、它回答什么问题、它和上下游怎么交接。**

## 为什么重要

很多流程问题表面上像工具问题，实际却是层级没分清：

- 把原始 reads 当成已经有坐标背景的数据；
- 把 BAM 当成可以独立解释的最终结果；
- 把 GTF/GFF 当成样本证据，而不是参考背景；
- 把 VCF 当成脱离 caller、参考版本和过滤逻辑的“最终真相”。

理解格式层级关系，有助于快速弄清：

- 当前分析停留在哪一层；
- 某个文件能回答什么问题；
- 某个结果为什么仍然依赖前一层的对象和背景。

## 各层之间如何交接

### 序列层：FASTA

这一层定义的是参考序列或组装序列本身。它提供后续定位、注释和解释的基本框架，但本身还不包含样本证据。

### 原始观测层：FASTQ

这一层持有的是实验直接产生的 reads 与质量值。它代表“样本观测”，但还没有被放进参考坐标系。

### 定位结果层：SAM / BAM / CRAM

这一层开始把 reads 放到参考背景下理解。此时我们拥有了位置、CIGAR、MAPQ 等信息，但还没有自动得到“生物学解释”。

### 注释与区间层：GTF / GFF / BED

这一层告诉我们参考上的某些区间对应什么对象或区域。它不是样本证据，而是解释定位结果和下游统计所依赖的参考背景。

### 变异结果层：VCF / BCF

这一层把定位证据进一步提炼成候选变异及其过滤信息。它比 BAM 更接近“结论”，但仍然依赖参考版本、caller 逻辑与上下文解释。

### 派生指标层：矩阵、coverage、汇总表

这一层常常是工作流交付给分析者的最终统计结果，例如表达矩阵、coverage 汇总、feature counts 等。它们通常已经离原始 reads 很远，因此解释时仍要能回溯到前面的层级。

## worked example

以一个简化的 DNA-seq 变异检测流程为例：

```text
FASTA -> FASTQ -> BAM -> VCF -> annotation / summary
```

这条链路里，每一步都不是简单“换格式”，而是在改变问题：

- `FASTA` 提供参考框架；
- `FASTQ` 提供样本的原始观测；
- `BAM` 表示这些观测如何被放到参考坐标系；
- `VCF` 是基于 BAM 与 caller 逻辑提炼出的候选变异结果；
- annotation / summary 则把变异继续接到数据库对象、临床解释或统计汇总层。

因此，如果你只看最后的 `VCF`，却不知道它来自什么参考版本、什么 BAM 证据和什么 caller 逻辑，就很容易误解结果。
