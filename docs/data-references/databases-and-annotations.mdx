# 数据库与注释系统一览

## 是什么

在 BioInfo 工作流中，我们经常会接触各种数据库和注释系统：

- 基因/转录本/蛋白注释：Ensembl, RefSeq, GENCODE；
- 变异数据库：dbSNP, ClinVar, gnomAD；
- 表达数据与公共数据集：GEO, SRA, TCGA 等。

它们各自有不同的数据模型、版本体系和命名方式，是“结果解释层”的基础背景。

## 注释系统：Ensembl, RefSeq, GENCODE 等

### Ensembl

- 提供基因组、基因、转录本、蛋白的大规模注释；
- 名称格式常为 ENSGxxxx（基因）、ENSTxxxx（转录本）等；
- 有版本号（release），支持多物种；
- 经常与 GENCODE 协作（在人类/小鼠上）。

### RefSeq

- NCBI 提供的参考序列集合；
- 使用 NM_/NR_/NP_ 等前缀；
- 更偏“精心整理的代表性序列”；
- 在很多临床解读工具和数据库中占重要地位。

### GENCODE

- 主要针对人类和小鼠的高质量基因/转录本集；
- 与 Ensembl 紧密集成；
- 常用于 RNA-seq 注释和转录本分析。

## 变异数据库：dbSNP, ClinVar, gnomAD 等

### dbSNP

- 历史悠久的 SNP/小变异数据库；
- 覆盖广泛，但质量和注释深度不一；
- 常用于“是否已知变异”的基础判断。

### ClinVar

- 人类临床变异数据库；
- 每条记录通常包含：
  - 变异位置、类型；
  - 与疾病的关联；
  - 临床意义（致病、良性、不明等）；
- 临床解读和一些研究常以 ClinVar 为重要参考。

### gnomAD

- 大规模人群变异频率数据库；
- 提供不同群体中的等位基因频率（AF）、覆盖度等信息；
- 常用于判断变异在一般人群中是罕见还是常见。

## 公共表达与队列数据：GEO, SRA, TCGA 等

- **GEO**：基因表达数据的公共存储库；
- **SRA**：测序读段（reads）的存档库；
- **TCGA** 等大型项目：提供统一处理过的队列数据和元信息。

在构建分析或对照结果时，可以：

- 从这些资源中找到相似实验或公共对照数据；
- 对比表达模式、变异频谱等。

## 版本与命名的重要性

不同数据库和注释系统有：

- 自己的版本号（release 版本、build）；
- 自己的 ID 命名规则（Ensembl ID、RefSeq ID 等）；

在整合数据时，需要注意：

- 确认 ID 类型和版本是否一致；
- 如有需要，使用映射表（mapping tables）或 BioMart 等工具进行 ID 转换；
- 在分析文档和结果文件中记录使用的数据库与版本。

## 实践建议

- 在项目 README 或 methods 部分明确写出：
  - 使用的参考基因组版本；
  - 使用的注释系统（Ensembl/GENCODE/RefSeq）及版本；
  - 使用的主要数据库及版本；
- 避免在同一分析中混用多个版本的注释或数据库，而不加区分；
- 对临床或高风险场景，优先选用维护良好且有清晰审稿流程的数据库。
