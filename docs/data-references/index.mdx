# 数据、注释与资源

这一部分把参考基因组、注释系统、数据库对象和常见文件格式放到同一层理解，因为它们共同决定了“输入输出到底是什么、结果最终如何解释”。

## 这一部分在全站中的位置

它连接对象层、方法层和应用层，是整站中最容易被低估、但最影响实际分析稳定性的板块之一。

## 为什么重要

很多流程问题并不是算法不会，而是：

- 参考版本不一致；
- 注释版本和坐标系统混乱；
- 文件格式层级没分清；
- 数据库对象无法正确映射到结果解释。

## 推荐阅读顺序

1. [参考基因组、坐标系统与注释](../foundations/reference-and-annotation.md)
2. [参考版本、构建号与 liftover](./reference-versions-and-liftover.mdx)
3. [数据库与注释系统一览](./databases-and-annotations.mdx)
4. [常见数据格式总览](./common-formats-overview.mdx)
