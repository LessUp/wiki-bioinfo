# 常见文件格式概览

## 是什么

生物信息学里的文件格式，不只是“文件后缀不同”。它们编码的是不同层次的对象：

- 原始观测（reads）
- 参考序列与注释
- 比对结果
- 特征计数
- 变异候选

理解这些格式，才能理解一个流程在每一步**真正持有的是什么信息**。

## 为什么重要

很多分析错误表面看起来像工具参数问题，实际却来自：

- 参考版本不一致；
- 坐标系统理解错误；
- 把 feature-level 文件当成 read-level 文件；
- 误解某个字段是否保留了多重比对、质量分数或注释信息。

## 核心格式与用途

### FASTA

用于保存参考序列或组装得到的 contig / scaffold。它只关心“序列本身是什么”，不保存质量值。

### FASTQ

用于保存原始测序 reads。它比 FASTA 多一层质量分数，因此是绝大多数测序工作流的起点。

### SAM / BAM

用于保存 reads 与参考序列的比对结果：

- SAM 是文本格式，便于检查；
- BAM 是其二进制压缩版本，更适合大规模分析。

这类文件通常保存：

- 比对位置；
- CIGAR；
- MAPQ；
- 配对信息；
- 额外标签。

### GTF / GFF

用于描述基因、转录本、外显子等注释特征。它回答的是“基因组上哪些区间对应哪些功能对象”。

### VCF

用于保存变异检测结果。它记录候选位点、参考 / 替代等位基因、质量信息和各种注释字段。
