# 常见数据格式

数据格式是把“生物问题”翻译成“计算输入输出”的接口层。

如果不知道一个流程读入什么、写出什么、字段各自代表什么，就很难真正理解工具在做什么，也很难排查错误来自哪一层。

## 这一部分在全站中的位置

这一节位于对象层、资源层和流程层之间：

- 它连接 [基础与数学](../foundations/index.md) 中的 reads、参考基因组、注释等对象；
- 它决定 [序列比对](../alignment/index.md)、[变异检测](../variants/index.mdx)、[转录组](../transcriptomics/index.mdx) 等流程模块如何交换数据；
- 它也影响你如何使用 [数据库与资源](../databases/index.md) 中的参考版本和注释文件。

## 为什么这一节重要

初学者最常见的问题不是不会跑命令，而是不知道：

- FASTQ 到底存了什么；
- BAM 和 SAM 的区别是什么；
- GTF / GFF 和 VCF 为什么不能混着理解；
- 一个流程为什么一定要指定参考基因组版本和注释版本。

理解数据格式，就是理解流程每一步的**输入、输出和约束**。
