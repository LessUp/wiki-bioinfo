# 概率、图与动态规划预备

## 是什么

如果把生物信息学看成一组计算问题，那么最常反复出现的抽象几乎可以归结为三类：

- **概率模型**：处理不确定性、噪声、状态转移和统计推断；
- **图模型**：处理连接关系、路径搜索、聚类结构和重复歧义；
- **动态规划**：处理具有最优子结构的问题，例如序列比对和状态解码。

这三类模型并不是彼此独立的“章节分类”，而是整个学科里反复穿插的共用语言。

## 为什么这三类模型反复出现

生物信息学数据通常具有三个特征：

1. **数据大且噪声高**：测序错误、覆盖不均、缺失观测都很常见；
2. **对象之间有结构关系**：序列、读段、转录本、进化树、网络都不是孤立点；
3. **很多任务本质是优化或推断**：我们要么寻找最优解释，要么寻找最可能解释。

概率、图和动态规划正好分别对应：

- 不确定性建模
- 结构建模
- 最优解计算

## 概率模型常见位置

概率模型通常出现在“不知道真实状态，只能从观测反推”的地方，例如：

- HMM 做基因预测或序列分段；
- motif 或 profile 模型描述位点偏好；
- 变异检测利用测序错误率和等位基因概率；
- 表达量估计依赖统计建模与归一化。

一个核心思想是：观测值并不等于真实状态，我们需要用概率去描述不确定性。

## 图模型常见位置

图模型适合处理“元素之间怎么连接”的问题，例如：

- de Bruijn graph 用于短读长组装；
- overlap graph 用于长读长组装；
- 系统发育树本身就是一种图结构；
- 蛋白互作网络和调控网络也可以表示为图。

在很多任务里，真正困难的地方不是单个节点，而是：

- 哪条边可信；
- 哪条路径合理；
- 重复和分叉该如何解释。

## 动态规划常见位置

动态规划适合处理“当前决策依赖更小子问题”的任务，典型例子包括：

- Needleman–Wunsch
- Smith–Waterman
- 编辑距离
- HMM 中的 Viterbi 解码

很多初学者会把动态规划只看作竞赛技巧，但在生物信息学中，它其实是很多精确算法的自然表达方式。

## 这三者如何互相连接

它们经常不是分开出现，而是叠在一起：

- HMM 既是概率模型，也常通过动态规划求最优状态路径；
- 组装问题既是图问题，也需要依赖统计信息去过滤错误边；
- 比对问题既是动态规划问题，也受到打分矩阵和错误模型影响。

因此，理解这三类模型的共同出现，比孤立背诵定义更重要。

## worked example

以 read mapping 为例：

- 用 **索引 / seed** 缩小候选区域；
- 用 **动态规划** 在候选区域内精细比对；
- 用 **概率或打分模型** 判断哪个比对结果最可信。

再看组装：

- 用 **k-mer** 把序列拆解；
- 用 **图** 表示连接关系；
- 用 **覆盖度和错误统计** 去清理图结构。

这说明真实工作流往往是多种模型的组合，而不是单一算法独立工作。

## 常见误区

### 图、概率、动态规划是互相独立的三门课

不是。它们在实际问题里经常同时出现。

### 只要会调用工具，就不需要理解这些模型

短期可能可以运行流程，但在解释结果、调参、排错时很快会遇到瓶颈。

### 动态规划只适合小数据

虽然精确二维 DP 在大规模场景下成本高，但其思想仍然深刻影响着大量实用工具的局部阶段和近似设计。
