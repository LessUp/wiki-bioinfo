# 参考基因组、坐标系统与注释

## 是什么

参考基因组是很多生物信息学流程默认的“共同坐标系”。

当我们说一个 read 被比到哪里、一个变异发生在哪个位置、一个 exon 覆盖了哪个区间时，本质上都在使用这套参考坐标系。

注释文件则在坐标之上进一步标记：

- 哪些区间对应基因；
- 哪些是 exon / intron / CDS；
- 哪些转录本属于同一个基因。

## 为什么重要

很多实际问题并不是算法不会，而是参考系统没对齐：

- 参考基因组版本和注释版本不匹配；
- 0-based / 1-based 坐标系统混淆；
- 某个结果使用了染色体命名规则，但另一个文件不兼容；
- 同一基因在不同数据库对象中的标识无法正确对应。

## 核心概念

### 参考基因组

它是一个被选定的序列框架，方便我们把 reads、变异、注释和功能信息放到统一背景下理解。

### 坐标系统

坐标系统决定“某个区间到底从哪里开始、到哪里结束”。不同格式和工具可能使用不同约定，因此转换时必须极其谨慎。

### 注释

注释告诉我们：在参考坐标上的不同区域分别对应什么生物学对象。没有注释，很多比对、定量和变异结果就很难解释。

## 常见误区

### 参考基因组就是“真实基因组”

不是。它是一个分析框架，不是某个个体或群体所有真实变化的完整代表。

### 只要序列版本对了，注释文件随便选都行

不对。注释版本与参考版本不匹配，会直接导致 feature 定位和结果解释错误。

### 坐标系统差 1 只是小问题

不是。对位点级分析来说，偏一个位置就可能完全改变结果解释。
