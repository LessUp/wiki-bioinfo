# 如何贡献

欢迎通过 Issue 和 Pull Request 参与建设 BioInfo Wiki。

## 适合贡献的内容

- 新知识点页面
- 对现有页面的纠错、补图、补例子
- 对术语、公式、图示的统一整理
- 对学习路线和交叉链接的改进
- 对站点样式、构建流程和自动化检查的优化

## 推荐贡献流程

1. 先开 Issue 或在已有 Issue 下认领任务。
2. Fork 仓库并新建分支。
3. 修改文档或站点代码。
4. 本地运行：

```bash
npm install
npm run start
npm run check
```

5. 提交 Pull Request，说明修改目标、影响范围和验证方式。

## 写文档时建议注意

- 优先解释概念之间的关系，不要只堆术语；
- 公式、图示、例子应为正文服务；
- 尽量说明方法的前提、优点和局限；
- 能给出交叉链接时尽量给出；
- 对不确定的内容请明确标注，而不是模糊表述。

## 内容质量标准

一篇合格页面至少应尽量满足：

- 有清晰的主题和读者目标；
- 有基本定义与背景；
- 有关键方法或模型解释；
- 有例子、图示或应用场景；
- 有参考资料；
- 有相关页面链接。
