# 写作规范

为了让 BioInfo Wiki 更接近 OI Wiki 的阅读体验，页面不仅要有知识点，还要尽量形成“可连续阅读、可回溯、可交叉引用”的结构。

## 推荐页面模板

普通概念页可以使用基础模板：

```md
# 标题

## 是什么
## 为什么重要
## 核心概念或模型
## 方法 / 算法 / 流程
## 例子或图示
## 应用场景
## 局限与注意事项
## 参考资料
## 相关页面
```

## 推荐算法页模板

对于编辑距离、比对、组装图、HMM 这类核心算法页，建议使用更完整的模板：

```md
# 标题

## 是什么
## 要解决什么生物信息学问题
## 数学或算法模型
## 核心递推 / 状态定义 / 图结构
## worked example（手算例子）
## 复杂度与适用前提
## 与真实工具或流程的连接
## 常见误区
## 参考资料
## 相关页面
```

## OI Wiki 风格的内容习惯

建议尽量补齐下面这些元素：

- **定义**：先说清楚问题是什么；
- **动机**：为什么会需要这个算法或模型；
- **形式化描述**：状态、递推、图模型、概率假设；
- **例子**：至少有一个简短 worked example；
- **复杂度/限制**：不要只说方法好，要说它在哪些条件下成立；
- **交叉链接**：把前置知识、后续知识和同类方法连起来；
- **工具映射**：适当说明该思想如何出现在真实软件或流程中。

## 术语规范

- 正文以中文为主；
- 关键术语首次出现时附英文，例如“隐马尔可夫模型（Hidden Markov Model, HMM）”；
- 数据库、软件和经典算法名称保留英文；
- 术语前后尽量保持一致，不要一页内混用多个译法。

## 图文规范

- 每张图都应该服务于一个具体解释目标；
- 图片文件建议放在 `static/img/illustrations/` 或未来按专题分目录管理；
- 图下注明图意，必要时说明数据来源或示意性质；
- 不要用装饰性图片代替解释。

## 公式与表格

- 需要公式时优先用 LaTeX 数学块清晰写出递推关系或概率定义；
- 表格适合用来比较全局比对/局部比对、PAM/BLOSUM、不同 `k` 取值的影响；
- 如果一个公式对后文非常关键，公式后要紧跟自然语言解释，不要只给公式不给含义。

## 代码与命令示例

- 命令要能直接运行，或明确指出需要替换的部分；
- 示例输入输出尽量简短；
- 如果命令依赖上下文，先交代数据格式和目标；
- 对算法页来说，伪代码或矩阵示意通常比冗长实现更重要。

## 链接规范

- 站内优先使用相对链接建立知识网络；
- 外链优先指向论文主页、官方文档或可信数据库；
- 引用教材和论文时尽量写明完整信息；
- 每个核心页面末尾都应有“相关页面”。
