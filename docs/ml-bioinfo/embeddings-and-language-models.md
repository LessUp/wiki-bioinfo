# 嵌入与语言模型

## 从字符串到表示学习

经典生物信息学常把序列直接作为字符串处理，而语言模型进一步尝试学习：

- 哪些片段经常共同出现；
- 哪些位置在上下文中“语义接近”；
- 一个序列在高维表示空间中的结构。

这样，每条序列都可以被映射成一个或多个向量表示（embedding），再用于聚类、检索、分类、生成或下游预测。

## 为什么语言模型适合生物序列

- 序列数据量极大，适合自监督预训练；
- 标签昂贵，但预训练可先学通用表示；
- 上下文依赖和远距离作用在蛋白和调控序列中都很重要。

## 常见对象

### 蛋白语言模型

这是目前最成熟的方向之一。模型在大规模蛋白序列数据库上预训练后，其 embedding 可用于：

- 功能注释；
- 突变效应预测；
- 结构相关任务；
- 相似蛋白检索。

### DNA / RNA 语言模型

这类模型尝试在更长、更重复、非均匀的信息背景中学习调控语法和结构模式，常见应用包括：

- 增强子 / 启动子分类；
- 剪接预测；
- 变异效应评分；
- 单细胞调控建模。

## 表示如何使用

语言模型可产生多层次表示：

- **token-level embedding**：每个位点 / 每个氨基酸位置一个向量；
- **sequence-level embedding**：整条序列一个汇总向量；
- **attention map**：提供一定程度的上下文关系线索。

这些表示可用于：

- 直接输入线性分类器；
- 用于相似性搜索；
- 与其他组学特征联合建模。

## 常见限制

- 预训练模型很大，推理成本高；
- embedding 的可解释性通常弱于经典 motif 模型；
- 不同任务是否真正受益于 foundation model，需要实证验证；
- 数据泄漏与评价设置不严谨时，容易高估性能。

## 常见误区

- **embedding 很像就说明功能一定一样**：只能说明表示空间接近，不等于功能完全相同；
- **预训练模型可以直接替代实验**：它们更像高质量先验和排序工具；
- **所有生物序列都该用 LLM**：很多问题用简单模型就够了。
