# k-means 在生物信息学中的应用

## 是什么

k-means 是一种经典的聚类方法，它要解决的问题是：

> 把一组对象分成 `k` 个簇，使同一簇内对象尽量接近，而不同簇之间尽量分开。

在生物信息学中，k-means 常被用于：

- 表达模式聚类；
- 样本分群；
- 初步发现潜在亚型或条件响应模式。

## 为什么它在表达数据里常见

对基因表达矩阵来说：

- 每个基因可以看成一个高维向量（在多个样本/时间点上的表达）；
- 我们常想找出“表达趋势相似的一组基因”；
- k-means 提供了一个很直接的目标：
  - 让每个对象靠近其簇中心（centroid）。

因此，它特别适合快速探索：

- 哪些基因有相似的时间序列变化；
- 样本是否能分成若干清晰亚群；
- 大矩阵中是否存在重复模式。

## 核心目标函数

k-means 的基本目标可以直观地理解为：

- 对每个对象，找到最近的簇中心；
- 让对象到簇中心的平方距离总和尽量小。

它通常通过“迭代两步”来做：

1. **Assignment step**：
   - 把每个对象分配给最近的中心；
2. **Update step**：
   - 用当前簇内对象的均值更新中心；

反复迭代，直到结果稳定。

## 在生物信息学中的典型用途

### 1. 基因表达模式聚类

把基因按表达趋势分组，寻找：

- 共表达模块；
- 某个时间过程中的共同上调/下调模式；
- 潜在的调控模块。

### 2. 样本聚类

在样本维度上，k-means 可以用来：

- 识别潜在亚型；
- 发现批次或污染问题；
- 给 PCA/UMAP 之后的低维表示做进一步分群。

## 局限

k-means 虽然常见，但有几个显著局限：

- 需要预先指定 `k`；
- 对初始化敏感；
- 更偏好“球形簇”；
- 在高维稀疏数据中，欧氏距离未必最合适；
- 不产生树结构，也不直接表达层次关系。

这也是为什么在 expression analysis 中：

- hierarchical clustering 和 k-means 常常一起出现；
- 前者更适合可视化层次，后者更适合快速得到离散分组。

## 与层次聚类的区别

- **Hierarchical clustering**：
  - 生成一棵 dendrogram；
  - 不需要预先指定簇数；
  - 更适合探索层次关系；
- **k-means**：
  - 直接输出 `k` 个簇；
  - 更适合快速形成扁平分组；
  - 对大矩阵更轻量。

因此，两者并不是“谁更高级”，而是各自适合不同目的。
