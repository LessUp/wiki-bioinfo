# 精确字符串匹配

## 是什么

精确字符串匹配要解决的问题是：

> 给定一个文本 `T` 和一个模式串 `P`，在 `T` 中找出所有与 `P` 完全相同的出现位置。

在生物信息学中，这个问题看似简单，但非常基础：

- 一个短 motif 是否出现在序列里；
- 一个 primer / probe 是否能完整匹配目标区域；
- 某段 reads 是否包含一个已知模式；
- 更复杂的近似匹配算法，常常也以它为起点。

## 朴素扫描

最直接的方法是：

- 把模式串从文本的每个可能起始位置开始尝试；
- 逐字符比较；
- 如果全部相同，则记录为一次匹配。

例如：

```text
T = A C G T A C G T
P = C G T
```

我们可以在 `T` 的每个位置对齐 `P`：

- 从位置 0 开始比较：首字符就失配；
- 从位置 1 开始：`CGT` 完全匹配；
- 继续向后扫描，直到找到所有出现位置。

这个方法的优点：

- 简单直观；
- 容易作为所有后续算法的基线；

但缺点是：

- 很多比较是重复的；
- 在长文本和大量查询下效率不高。

## 为什么需要更聪明的算法

考虑这种情况：

```text
T = A A A A A A A A ...
P = A A A A B
```

朴素扫描会在很多位置重复比较前面的 `AAAA`，然后在最后一个字符 `B` 才失配。

也就是说：

- 我们已经学到一些“前缀结构”信息；
- 但朴素方法没有利用这些信息；
- 更好的算法会想办法在失配后避免无谓的重复比较。

## 算法设计上的核心问题

精确匹配中的关键问题不是“如何比较字符”，而是：

- 当发生失配时，下一次从哪里继续；
- 能否利用模式串本身的重复结构；
- 能否一次处理多个模式；
- 能否把搜索建立在预处理索引之上。

这也是为什么精确匹配会自然延伸到：

- failure function / 回退策略；
- trie / 多模式匹配；
- suffix tree / suffix array；
- BWT / FM-index。

## 在生物信息学中的位置

精确匹配本身有直接用途：

- exact k-mer matching；
- primer / barcode 校验；
- motif 搜索中的固定串情况；

同时它也为更复杂问题提供基础直觉：

- 近似匹配是在此基础上允许 mismatch / gap；
- seed-and-extend 常先做精确 seed 匹配，再进入局部扩展；
- suffix-based 索引正是在“快速做大量 exact matching”的需求下发展起来的。

## 常见误区

### 精确匹配太简单，不值得单独讨论

并不是。很多更复杂的字符串算法，正是围绕“如何比朴素扫描更高效地做 exact matching”发展出来的。

### 近似匹配会完全取代精确匹配

不对。精确匹配仍然是很多 read mapper、motif 扫描和索引结构中的第一步。
