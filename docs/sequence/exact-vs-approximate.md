# 精确匹配与近似匹配

## 是什么

精确匹配（exact matching）要求 pattern 和参考中的某个片段完全一致；近似匹配（approximate matching）则允许错配、插入缺失或更一般的编辑代价。

这两个问题看起来只差“一点误差容忍”，但在算法设计上其实差异很大。

## 要解决什么生物信息学问题

真实生物序列和参考序列之间几乎不可能总是完全相同，因为会存在：

- 测序错误；
- 生物学变异；
- 参考与样本的天然差异；
- 插入缺失和重复区域。

所以 read mapping 的一个核心难题是：

- 如何先快速找到候选位置；
- 再判断这些候选在允许误差时是否仍然合理。

## 输入与输出

- 输入：query 序列、参考序列或其索引
- 输出：完全匹配位置，或在给定误差模型下的候选匹配位置

## 核心思想

### 精确匹配

适合使用 suffix array、BWT、FM-index 或哈希索引等结构快速完成查找。它的优势是快、结构清晰，但无法直接容忍真实数据中的偏差。

### 近似匹配

常通过动态规划、seed-and-extend、带约束搜索等方式实现。它更贴近真实问题，但计算通常更贵。

## 为什么这一区别重要

现代比对工具常常不会在全参考上直接做昂贵的近似匹配，而是：

1. 先利用精确匹配快速找到 seeds；
2. 再对候选位置做局部扩展；
3. 最后结合错配、gap 和评分体系做判断。

也就是说，真实 read mapping 往往是“精确匹配 + 近似匹配”的组合，而不是二选一。

## 常见误区

### 既然真实数据有错误，精确匹配就没有意义

不对。精确匹配常用于候选搜索，是现代工具提速的关键。

### 近似匹配只是把精确匹配稍微放宽一点

也不准确。允许错误后，问题复杂度、状态空间和搜索策略都会显著变化。
