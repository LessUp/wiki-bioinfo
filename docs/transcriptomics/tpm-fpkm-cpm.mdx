# TPM、FPKM、CPM 与有效长度

## 是什么

在 RNA-seq 中，“表达量”常用不同单位表示：

- **counts**：原始 read 或 fragment 计数；
- **CPM**（counts per million）：按测序深度归一化的计数；
- **FPKM**（fragments per kilobase per million）：按转录本长度和测序深度归一化；
- **TPM**（transcripts per million）：先按长度归一化，再按样本内总量归一化。

这些单位背后隐含了不同的假设和“比较方式”，理解它们的关系有助于解释表达矩阵。

## 为什么不直接看 counts

原始 counts 受多种因素影响：

- 样本的总测序深度；
- 基因/转录本的长度；
- 文库复杂度和技术噪声；

如果直接比较 counts：

- 长基因天然得到更多 reads；
- 深度更高的样本整体 counts 更大；

因此，需要先做至少两类归一化：

1. 按测序深度归一化（library size）；
2. 按序列长度归一化（length）。

## CPM

**CPM（counts per million）** 是最简单的一种：

- 先计算每个样本的总计数；
- 再把每个基因的 counts 除以总计数并乘以一百万。

适合用在：

- 粗略比较同一任务中不同基因的表达；
- 做简单过滤（例如去掉 CPM 太低的基因）。

局限在于：

- 没有考虑基因长度；
- 仍然对组成偏差敏感（少数高表达基因会干扰整体分布）。

## FPKM

**FPKM** 试图进一步把基因长度纳入考虑：

- 先按长度（kb）对 counts 做归一化；
- 再按测序深度（per million fragments）归一化。

直观上，它回答的是：“在同样深度下，单位长度上的表达有多高？”

但实践中，人们发现 FPKM 在样本间比较时存在一些问题：

- 不同样本的 scaling 因子不容易对齐；
- 在高表达基因导致的组成偏差下，解释变得困难。

## TPM

**TPM（transcripts per million）** 改变了归一化顺序：

1. 先按长度对每个转录本的 counts 做归一化；
2. 再在样本内把所有归一化值加总并 rescale 到一百万。

这样带来的好处是：

- 不同样本之间的 TPM 总和相同（每个样本总 TPM 为一百万）；
- 更易于跨样本对比表达份额（相对量）。

TPM 常被认为比 FPKM 更稳定，适合作为“表达占比”的度量。

## 有效长度（effective length）

在真实 RNA-seq 数据中，有效能够产生 reads 的长度往往不是简单的转录本长度：

- read 长度；
- 序列两端不能完整覆盖的区域；
- 某些区域质量低或难以比对；

都会影响“单位长度被抽样的概率”。

因此很多定量方法会使用**有效长度（effective length）**：

- 通常是转录本长度减去与 read 长度有关的校正项；
- 有时还考虑 GC 内容或结构等因素。

这让基于 TPM 的比较更加接近“真实可观测的表达水平”。

## 实践中的选择

- 做下游差异分析时，通常使用的是**原始 counts**（配合合适的归一化和统计模型），而不是直接在 TPM 上做检验；
- 做可视化和样本间粗略比较时，更常使用 TPM 或 log-TPM；
- CPM 常用于过滤或简单 QC。

## 注意事项

- 不要在没有说明的情况下混用 FPKM、TPM 和 CPM；
- 差异分析通常不要直接用 TPM 做 t-test，而是用专门的 count 模型；
- 解释结果时，要区分“相对表达”（TPM）和“统计检验用的输入”（counts）。
