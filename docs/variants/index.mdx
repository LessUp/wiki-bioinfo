# 变异检测

<PageHeaderMeta
  section="Applications"
  audience="已经理解比对与参考基因组，希望把方法放回 DNA-seq 任务中的读者"
  startWith="先看 reads/coverage、常见文件格式与序列比对，再进入 variant calling"
/>

变异检测关注的是：给定样本 reads 与参考基因组，如何识别 SNP、InDel 乃至更复杂的结构变异，并判断这些差异是否足够可信、是否具有生物学意义。

## 这一部分在全站中的位置

这一节位于“分析方向与案例”之下，承接序列比对、参考基因组、错误模型与文件格式，最终把这些方法放回 DNA-seq 任务中。

## 为什么这一节重要

很多读者会把 variant calling 理解成“跑一个 caller 然后看 VCF”。

但真正关键的问题是：

- 证据来自哪些 reads；
- 比对不确定性如何影响判断；
- 测序错误和真实变异如何区分；
- 过滤标准、重复区域和参考版本如何改变结果解释；
- small variants 与结构变异、germline 与 somatic 之间如何区分和建模。
