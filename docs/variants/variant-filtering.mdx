## 是什么

DNA-seq 变异过滤（variant filtering）是在 caller 产生一批候选变异之后，对这些候选根据证据强度和上下文进行筛选的过程。

它的目标不是“找到所有差异”，而是：

- 尽量保留可信的生物学变异；
- 尽量剔除由测序错误、比对错误或模型局限引起的假阳性；
- 在召回率和精确率之间做合理平衡。

## 常用字段直觉

不同 caller 会引入更多字段（如 MQ、QD、FS、SOR 等），但大体都在描述：

- 证据强度（深度、质量）；
- 信号是否偏好某一条链或某一方向；
- 变异是否位于已知问题区段（低复杂度、重复、黑名单区域）。

## Hard filter 与模型化过滤

在实践中，经常会看到两类过滤方式：

### 1. Hard filter

使用一组阈值对字段进行直接裁剪，例如：

- `DP < 某个阈值`：深度太低，不可信；
- `QUAL < 某个阈值`：整体质量偏低；
- 某些 caller 还会基于 QD（QUAL/DP）、FS（strand bias）给出硬阈值建议。

优点：

- 简单直观；
- 便于解释；

缺点：

- 对不同测序平台和实验条件不够自适应；
- 很难兼顾边缘情况和复杂区域。

### 2. 模型化过滤

一些 caller 会使用更复杂的统计或机器学习模型估计“该位点是真实变异”的概率，并在 FILTER 字段中标记不同的 failure 原因。

优点：

- 可以结合多个特征（深度、偏好性、mapping 质量等）；
- 更有机会适应不同数据特征；

缺点：

- 行为更黑箱，调参和解释难度增加；
- 跨项目迁移时需要重新理解模型假设。

## 一个简化的过滤思路示例

在教学/探索阶段，可以先用一个较为保守的简化思路：

- **基础要求**：
  - 深度 `DP` 不要太低（例如 &gt;= 10，具体视实验而定）；
  - MAPQ 过低的 reads 所支撑的变异要格外小心；
- **看 FILTER**：
  - 优先关注 `FILTER = PASS` 的位点；
  - 对被特定标记剔除的位点（如 strand bias、低质量）结合字段说明与 caller 文档理解原因；
- **看 AF 与 read 支持**：
  - 极端 AF（过低或过高）需要结合深度与覆盖度背景解释；
  - 小心只有少数 reads 支持、且这些 reads 自身质量或 mapping 不佳的位点。

需要强调的是：

> 不存在“一组对所有数据都正确”的通用阈值。

过滤策略一定要结合：

- 测序平台与实验设计；
- caller 的具体假设和字段含义；
- 下游分析（如群体分析、诊断、筛查）的容错与风险偏好。

## 与重复区域和低复杂度区域的关系

即使字段看起来 "不错"，在重复序列和低复杂度区域，以下风险依然存在：

- 多重比对导致 reads 定位位置不稳定；
- mapping 错位导致假阳性 indel 或 SNP；
- coverage 异常堆积掩盖真实的变异模式。

因此，variant filtering 不应脱离：

- 参考基因组注释（重复、GC、黑名单区域等）；
- mapping 质量（MAPQ、多重比对标签）；
- 区域上下游其他变异和 coverage 模式。

<PitfallsBox
  items={[
    '仅凭单个字段（例如 QUAL）做一刀切过滤，而不看深度、偏好性和 mapping 质量。',
    '把某套阈值不加调整地复制到不同平台、不同实验条件甚至不同 caller 上。',
    '忽略重复或低复杂度区域的特殊性，把这些区域的 PASS 变异与普通区域一视同仁。',
