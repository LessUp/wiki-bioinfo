## 是什么

BLAST（Basic Local Alignment Search Tool）是一族用于在大型序列数据库中寻找**局部高相似片段**的工具。

它更像是一种搜索框架，而不是单一算法实现：

- 先用较轻量的方式找到候选局部匹配（seed / word hits）；
- 再在这些候选附近做更精细的扩展和得分；
- 输出按得分和统计意义排序的局部比对结果。

## 要解决什么生物信息学问题

经典问题是：

> 给定一条 query 序列，在一个很大的数据库里找到与之局部相似的片段，并评估这些相似性是否可能“超出随机期望”。

典型应用包括：

- 根据序列相似性寻找可能的同源基因；
- 在蛋白数据库中查找保守结构域；
- 做快速注释或功能推断的粗筛。

## 输入与输出

- **输入**：
  - 一条或多条 query 序列；
  - 一个已索引的序列数据库（核酸或蛋白）；
  - 打分矩阵（如 BLOSUM/ PAM）和 gap 罚分设置；
- **输出**：
  - 局部比对片段（HSPs）及其得分；
  - 统计指标（如 E-value）；
  - 可视化比对格式（带匹配/错配、gap 的局部 alignment 片段）。

## 核心思想：seed-and-extend 在 BLAST 中的具体化

BLAST 的核心是一种典型的 seed-and-extend 流程：

1. **构造 word 列表**：
   - 把 query 序列按固定长度 `w` 划分成一系列 word（例如长度 3 的氨基酸片段）。
2. **词表扩展与过滤**：
   - 基于打分矩阵生成一组“与这些 word 相似且得分不太低”的 candidate words；
3. **在数据库中快速定位这些 word**：
   - 利用预构建的索引在数据库中找到这些 words 的所有出现位置；
4. **从 hits 处开始局部扩展**：
   - 在 word 命中的位置附近向两侧扩展 alignment，直到得分不再有增长空间或跌破阈值；
5. **合并/过滤并排序**：
   - 把显著的局部比对片段收集起来，估计其随机出现的概率（得分分布 / E-value），再排序输出。

相比“在整个数据库上做完整二维 DP”的做法，BLAST 使用 seed-and-extend 极大降低了计算成本，却仍保留了对局部高相似片段的敏感性。

## BLAST 系列变体

常见变体包括：

- **blastn**：核酸 vs 核酸；
- **blastp**：蛋白 vs 蛋白；
- **blastx / tblastn / tblastx**：结合翻译框架的跨层次搜索；
- 以及 BLAST+ 工具集中的各种实现形态。

核心思想相似，但：

- word 长度选择；
- 打分矩阵；
- 搜索空间与统计模型

会随任务类型而调整。

## 复杂度与适用前提

BLAST 牺牲了一部分“理论上的最优敏感性”，换取可接受的计算成本。适用前提通常是：

- 数据库规模较大；
- 允许略过部分非常弱的相似性；
- 关注的是局部片段的同源性，而非全长比对。

对于更小的数据集或需要更精确局部比对的场景，Smith–Waterman 等方法仍然有价值，只是计算更贵。

## 与全局/局部比对和 seed-and-extend 的关系

- 在“目标函数”层面，BLAST 更接近 **局部比对**：它寻找的是局部高分片段；
- 在“实现策略”层面，它采用的是 seed-and-extend 型结构：
  - seed 阶段负责缩小候选空间；
  - extend 阶段负责精细局部 alignment。

因此，可以把 BLAST 看成是：

> Smith–Waterman 式局部比对的“工程化近似版 + 统计评估 + 数据库搜索框架”。

## 与真实工具或流程的连接

在现代分析流程中，BLAST 仍然是：

- 做注释粗筛、功能预测的常用工具；
- 在教学和方法对比中解释 seed-and-extend 思想的经典例子。

但对于 NGS 规模的 read mapping，现代工具（如 BWA、Bowtie、minimap2）通常使用更适合短/长读长的索引和 seeding 策略。

## 常见误区

- **把 BLAST 当成全局比对工具**：
  - 实际上它输出的是局部 alignment，不能简单理解为“两个序列整体相似到什么程度”。
- **只看百分比 identity，不看 E-value/得分**：
  - 在大数据库里，即使 identity 较高，也可能是随机出现，需要结合得分分布解释。
- **在 NGS read mapping 中直接使用 BLAST**：
  - 计算成本通常难以接受，且很多 read-level任务已有更合适的专门 mapper。
