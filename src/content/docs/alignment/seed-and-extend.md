## 是什么

Seed-and-extend 是现代序列搜索与 read mapping 的核心策略：

```
┌─────────────────────────────────────────┐
│  Step 1: Seeding（锚定）                │
│  快速找到可靠的局部精确匹配片段          │
│         ↓                               │
│  Step 2: Extension（扩展）                │
│  在候选位置周围进行精细比对              │
└─────────────────────────────────────────┘
```

核心思想：将"全局搜索近似匹配"分解为"先缩小候选范围，再局部精细求解"。

## 要解决什么生物信息学问题

在全基因组参考上逐条比对 read 的完整近似匹配，计算代价极高。

**Seed-and-extend 的解决方案：**

| 步骤 | 方法 | 效果 |
|-----|------|------|
| Seeding | k-mer / minimizer / FM-index | 快速锁定候选位置 |
| Extension | 局部动态规划 | 仅在少数区域进行昂贵计算 |
| 整体 | 保留容错能力 | 容忍 mismatch、indel 等变异 |

## 输入与输出

- 输入：query 序列、参考索引、seeding 规则、扩展评分模型
- 输出：候选比对位置及其局部最优对齐结果

## 核心思想

**典型流程：**

```
输入: query read, 参考基因组索引
   │
   ▼
┌────────────────┐
│  1. 精确匹配   │  ← 使用 k-mer / minimizer / FM-index
│     (seeding)  │     寻找精确匹配的 seed
└────────────────┘
   │
   ▼
┌────────────────┐
│  2. 候选过滤   │  ← 根据 seed 数量、位置关系、
│                │     链方向等过滤假阳性
└────────────────┘
   │
   ▼
┌────────────────┐
│  3. 局部扩展   │  ← 局部动态规划（如 Smith-Waterman
│   (extension)  │     或带 band 的 DP）
└────────────────┘
   │
   ▼
输出: 比对位置、得分、CIGAR
```

## Worked Example

**场景：** 100bp read，中间有 2 个错配和 1 个 3bp 插入

```
Read:    ████████████████████████████████████
          ↓错配    ↓3bp插入    ↓错配
Ref:     ████████-------███████████████████

传统精确匹配:  ❌ 全长精确匹配失败（差异 > 阈值）

Seed-and-extend:
  ├─ Seed 1: 位置 10-20 精确匹配 ✓
  ├─ Seed 2: 位置 50-60 精确匹配 ✓  
  └─ Extension: 连接两个 seed，容忍中间差异 ✓
```

**关键洞察**：只要有足够长的精确匹配片段（seed），就能在扩展阶段恢复完整比对。

## 复杂度与适用前提

**优势：**
| 特性 | 说明 |
|-----|------|
| 搜索快 | seeding 阶段通常 O(n) 或次线性 |
| 扩展局部化 | DP 仅在候选区域进行 |
| 容错性强 | 能处理测序误差和真实变异 |

**Seed 设计的权衡：**

| seed 长度 | 影响 | 结果 |
|----------|------|------|
| 太短 | 假阳性多 | 扩展阶段负担重 |
| 太长 | 敏感性低 | 易错过弱相似匹配 |
| 特殊区域 | 低复杂度/重复区 | seeding 困难 |

**现代工具的策略**：使用 multiple seeds、adaptive seeding、chaining 等技术平衡敏感性和特异性。

## 与真实工具或流程的连接

从 BLAST 到很多 read mapper，这种“先找 seed、再扩展”的思想都非常常见。它是把索引结构和动态规划结合起来的典型工程路线。

## 常见误区

### 只要有 seed 就一定能恢复正确比对

不一定。seed 的质量、重复区域、错误分布和扩展模型都会影响最终结果。

### extend 只是补一下边角，不重要

不是。真正决定 gap、错配和局部最优路径的往往就是扩展阶段。
