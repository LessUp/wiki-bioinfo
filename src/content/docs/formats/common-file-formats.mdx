## 是什么

生物信息学里的文件格式，不只是“文件后缀不同”。它们编码的是不同层次的对象：

- 原始观测（reads）
- 参考序列与注释
- 比对结果
- 特征区间
- 变异候选
- 派生指标

理解这些格式，才能理解一个流程在每一步**真正持有的是什么信息**。

## 为什么重要

很多分析错误表面看起来像工具参数问题，实际却来自：

- 参考版本不一致；
- 坐标系统理解错误；
- 把 feature-level 文件当成 read-level 文件；
- 误解某个文件是否保留了质量分数、位置信息或注释背景。

换句话说，文件格式是流程中最容易被忽略、但最直接影响解释边界的一层。

## 各类格式到底装了什么

### FASTA

**保存什么：**

- 参考序列本身（基因组、转录本、蛋白、contig/scaffold 等）；
- 序列 ID 与可选描述信息。

**不保存什么：**

- read-level 质量值；
- 比对位置；
- coverage 或变异证据。

**常见上下游关系：**

- 常作为 mapping、assembly、motif 分析或序列提取的参考输入；
- 经常和 GTF/GFF 一起出现，但两者表达的是不同对象层。

### FASTQ

**保存什么：**

- 原始测序 reads；
- 每个碱基的质量值。

**不保存什么：**

- read 在参考上的位置；
- 转录本、基因或变异解释；
- 经过过滤后的 feature-level 统计。

**常见上下游关系：**

- 往下游常接 QC、比对、pseudo-alignment 或 assembly；
- 如果只有 FASTQ，你还没有任何参考坐标或注释层解释。

### SAM / BAM / CRAM

**保存什么：**

- reads 与参考序列之间的定位结果；
- 位置、CIGAR、MAPQ、flag、配对信息及可选标签。

**不保存什么：**

- 独立于参考之外的完整“真相”；
- 下游注释数据库对这些位置的解释；
- 自动保证正确的参考版本背景。

**常见上下游关系：**

- 上游通常来自 FASTQ + 参考序列索引；
- 下游常进入变异检测、可视化、coverage 统计、feature counting 等环节。

### GTF / GFF

**保存什么：**

- gene、transcript、exon、CDS 等注释特征；
- 坐标、链方向、feature 类型与属性字段。

**不保存什么：**

- 原始 reads；
- 比对证据；
- 已经聚合好的表达矩阵或变异结果。

**常见上下游关系：**

- 常与参考 FASTA 搭配解释参考上的功能对象；
- 在 RNA-seq 定量、feature counting 与转录本分析中经常是关键背景文件。

### BED

**保存什么：**

- 简洁的区间坐标集合；
- 在很多场景下是“区域列表”的轻量表示。

**不保存什么：**

- 完整注释对象层级关系；
- 丰富的 gene / transcript 属性体系；
- 原始测序证据。

**常见上下游关系：**

- 常用于 peak、target region、窗口、候选区间或浏览器轨道；
- 适合做区间运算，但不应自动当成等价注释文件使用。

### VCF / BCF

**保存什么：**

- 候选变异位置与 REF / ALT；
- QUAL、FILTER、INFO 以及 genotype 等字段；
- caller 和注释流程附加的结果信息。

**不保存什么：**

- 产生这些变异调用的全部 read-level 证据；
- 自动保证正确的参考版本、注释版本和过滤逻辑解释；
- 独立于分析上下文的最终结论。

**常见上下游关系：**

- 上游依赖 BAM/CRAM、参考序列和 caller 逻辑；
- 下游常进入过滤、注释、数据库映射与结果解释。

## worked example

假设你收到一个分析交付包，里面有：

- `reference.fa`
- `genes.gtf`
- `sample.bam`
- `sample.vcf.gz`

一种正确的阅读方式不是把它们都看成“样本结果文件”，而是先判断它们分别位于哪一层：

- `reference.fa`：提供参考序列框架；
- `genes.gtf`：说明参考坐标上哪些区域对应 gene / transcript / exon；
- `sample.bam`：保存 reads 被放到参考上的定位结果；
- `sample.vcf.gz`：保存基于 BAM 与 caller 逻辑得到的候选变异结果。

这也意味着：

- 只有 `sample.vcf.gz`，你很难脱离参考版本与过滤逻辑解释结果；
- 只有 `sample.bam`，你还没有直接得到“最终变异结论”；
- 只有 `reference.fa` 和 `genes.gtf`，你只有背景，没有样本证据。

这个例子说明：这些文件不是互相替代，而是在同一个流程里承担不同层的职责。

## 常见误区

<PitfallsBox
  items={[
    'FASTA 和 FASTQ 都装序列，所以可以互相替换。实际上前者更偏参考序列，后者保存的是 reads 与质量值。',
