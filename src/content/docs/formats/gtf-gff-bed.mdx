## 是什么

这三种格式都编码**基因组上的位置信息**，但抽象层次不同：

| 格式 | 编码对象 | 抽象层次 | 典型用途 |
|------|---------|---------|---------|
| **GTF/GFF** | 基因结构注释 | 层级化（gene → transcript → exon） | RNA-seq 定量、变异功能注释 |
| **BED** | 区间集合 | 扁平（无层级关系） | Peak 区域、靶标 panel、CNV 区间 |

## GTF / GFF：层级化注释结构

### GTF 格式（Gene Transfer Format）

九列制表符分隔：

```text
chr1  HAVANA  gene  11869  14409  .  +  .  gene_id "ENSG00000223972"; gene_type "transcribed_unprocessed_pseudogene";
```

| 列号 | 字段 | 说明 |
|------|------|------|
| 1 | seqname | 参考序列名称（chr1） |
| 2 | source | 注释来源（HAVANA、ENSEMBL） |
| 3 | feature | 特征类型（gene、transcript、exon、CDS） |
| 4 | start | 1-based 起始坐标 |
| 5 | end | 1-based 终止坐标 |
| 6 | score | 置信度（`.` 表示无） |
| 7 | strand | 链方向（`+` / `-` / `.`） |
| 8 | phase | 阅读框相位（0/1/2，仅 CDS 有效） |
| 9 | attributes | 键值对属性（gene_id、transcript_id 等） |

### GFF3 格式（General Feature Format v3）

与 GTF 类似，但：
- 支持更灵活的层级关系（通过 `Parent` 属性）
- 属性使用 `=` 分隔（GTF 使用空格）
- 支持任意特征类型

```text
chr1  HAVANA  gene  11869  14409  .  +  .  ID=ENSG00000223972;Name=DDX11L1
chr1  HAVANA  mRNA  11869  14409  .  +  .  ID=transcript:ENST00000;Parent=ENSG00000223972
chr1  HAVANA  exon  11869  12227  .  +  .  Parent=transcript:ENST00000
```

### 层级结构示例

```text
gene: ENSG00000223972 (chr1:11869-14409)
  ├── transcript: ENST000001
  │     ├── exon: 11869-12227
  │     ├── exon: 12613-12721
  │     └── exon: 13221-14409
  └── transcript: ENST000002
        ├── exon: 11869-12227
        └── exon: 12900-14409
```

## BED：扁平区间集合

### 基本格式（3 列必需）

```text
chr1  1000  5000
chr2  300   800
```

### 扩展格式（最多 12 列）

```text
chr1  1000  5000  BRCA1_peak  950  +  1000  5000  255,0,0  2  1500,1200  0,2800
```

| 列号 | 字段 | 必需？ | 说明 |
|------|------|--------|------|
| 1 | chrom | ✓ | 染色体 |
| 2 | start | ✓ | 0-based 起始 |
| 3 | end | ✓ | 终止（half-open） |
| 4 | name | ✗ | 区间名称 |
| 5 | score | ✗ | 分数（0-1000） |
| 6 | strand | ✗ | 链方向 |
| 7 | thickStart | ✗ | 显示起始（如 CDS） |
| 8 | thickEnd | ✗ | 显示终止 |
| 9 | itemRgb | ✗ | RGB 颜色 |
| 10 | blockCount | ✗ | 外显子/块数量 |
| 11 | blockSizes | ✗ | 各块大小（逗号分隔） |
| 12 | blockStarts | ✗ | 各块起始（逗号分隔） |

### 典型应用场景

- **ChIP-seq peaks**：`chr1  1000  1500  peak_1  950`
- **Exome panel**：靶标区域列表
- **CNV regions**：拷贝数变异区间
- **Windowing**：基因组分窗统计

**这是最常见的错误来源之一**。混淆坐标系统会导致区间偏移 1 bp，可能使 peak 注释错误或变异功能判断失误。

## 常见概念误区

<PitfallsBox
  items={[
    '**坐标系统混淆**：直接将 GTF 的 start 当作 BED 的 start 使用，导致区间偏移 1 bp。',
    '**GTF 与 BED 功能等价**：两者虽然都表示"区间"，但 GTF 是层级注释结构，BED 是扁平区间列表，语义不等价。',
    '**GTF = GFF**：GTF 是 GFF2 的严格子集，但 GFF3 有更灵活的属性定义，工具兼容性不同。',
    '**注释版本不匹配**：使用 GRCh37 的 BAM 配合 GRCh38 的 GTF，导致坐标无法映射。',
    '**phase 字段误用**：phase（0/1/2）仅对 CDS 特征有效，gene/exon 特征应为 `.`。',
  ]}
/>

## 本章小结

- GTF/GFF 编码**层级化基因组注释**（gene → transcript → exon）
- BED 编码**扁平区间集合**（peaks、panel、CNV）
- GTF 使用 **1-based closed** 坐标，BED 使用 **0-based half-open** 坐标
- 格式选择取决于下游工具需求和区间运算类型
