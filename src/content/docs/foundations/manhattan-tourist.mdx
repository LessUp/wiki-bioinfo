## 问题模型

给定一个 $n \times m$ 的网格，每条水平边和垂直边都有一个权重（景点数）。只能向右或向下移动。

**目标**：找到一条从左上角 $(0,0)$ 到右下角 $(n,m)$ 的路径，使路径上边权重之和最大。

## 动态规划分析

### 状态定义
设 $s_{i,j}$ 为到达交叉点 $(i,j)$ 时能收集到的最大景点数。

### 递推关系
要到达 $(i,j)$，只能从上方 $(i-1,j)$ 或左方 $(i,j-1)$ 过来：

$$s_{i,j} = \max \begin{cases} s_{i-1,j} + \text{weight}(\text{vertical edge to } i,j) \\ s_{i,j-1} + \text{weight}(\text{horizontal edge to } i,j) \end{cases}$$

### 初始化
- $s_{0,0} = 0$
- 第一行只能从左边推导：$s_{0,j} = s_{0,j-1} + w_{\text{horizontal}}$
- 第一列只能从上面推导：$s_{i,0} = s_{i-1,0} + w_{\text{vertical}}$

## 算法伪代码

```text
MANHATTANTOURIST(down_weights, right_weights, n, m)
1  s[0,0] ← 0
2  for i ← 1 to n
3      s[i,0] ← s[i-1,0] + down_weights[i,0]
4  for j ← 1 to m
5      s[0,j] ← s[0,j-1] + right_weights[0,j]
6  for i ← 1 to n
7      for j ← 1 to m
8          s[i,j] ← max(s[i-1,j] + down_weights[i,j], 
                        s[i,j-1] + right_weights[i,j])
9  return s[n,m]
```

## 从网格到有向无环图 (DAG)

曼哈顿是一个规则网格，但现实中（如百老汇大道纵切曼哈顿）或生物学问题中，路径可能更复杂。我们可以将任何网格或比对图抽象为 **DAG (Directed Acyclic Graph)**。

### 核心概念
- **入度 (Indegree)**：进入一个顶点的边数。
- **出度 (Outdegree)**：从一个顶点出去的边数。
- **前驱 (Predecessors)**：所有能直接到达顶点 $v$ 的顶点集合 $P(v)$。

### DAG 中的最长路径递推
对于任意顶点 $v$，其得分 $s_v$ 为：
$$s_v = \max_{u \in P(v)} \{ s_u + \text{weight}(u, v) \}$$

### 拓扑排序 (Topological Ordering)
为了确保在计算 $s_v$ 时，其所有前驱 $s_u$ 已经计算完毕，我们需要按照**拓扑排序**的顺序遍历顶点。
- 在规则网格中，按行、按列或按对角线遍历都是合法的拓扑排序。
- 在通用 DAG 中，拓扑排序保证了所有边的方向都是从序号小的顶点指向序号大的顶点。

## 为什么这一模型至关重要？
曼哈顿旅游问题几乎就是**序列比对**（Sequence Alignment）的代名词：
1. **顶点**：代表两个序列中碱基的交叉点 $(i, j)$。
2. **水平/垂直边**：代表插入或删除 (Indel/Gap)。
3. **对角线边**：代表两个碱基的匹配 (Match) 或错配 (Mismatch)。
4. **权重**：代表得分系统（如 Match=+1, Mismatch=-1, Gap=-2）。

通过构建不同的“曼哈顿网格”，我们可以灵活地解决全球比对、局部比对、重叠比对等多种生物信息学核心任务。
