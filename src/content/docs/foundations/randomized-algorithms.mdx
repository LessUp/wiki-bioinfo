## 为什么需要随机化算法

确定性算法在处理某些问题时会遇到困难：

- **陷入局部最优**：贪心算法可能被困在局部最优解
- **算法复杂**：确定性算法可能需要复杂的分析
- **最坏情况差**：某些确定性算法的最坏情况性能很差

随机化算法通过引入随机性解决这些问题：

## 1. 快速排序的随机化

### 确定性快速排序的问题

确定性快速排序选择第一个元素作为 pivot：

- **最坏情况**：已排序数组，复杂度 $O(n^2)$
- **平均情况**：$O(n \log n)$
- **问题**：实际数据可能接近已排序，触发最坏情况

### 随机化快速排序

随机选择 pivot：

```
RANDOMIZEDQUICKSORT(array, low, high)
1 if low < high
2   pivot_index ← RANDOM(low, high)
3   swap array[low] and array[pivot_index]
4   pivot ← partition(array, low, high)
5   RANDOMIZEDQUICKSORT(array, low, pivot - 1)
6   RANDOMIZEDQUICKSORT(array, pivot + 1, high)
```

### 复杂度分析

**期望时间复杂度**：$O(n \log n)$

**关键观察**：
- 随机选择 pivot 使得每个元素有相等的概率成为 pivot
- 期望的递归深度是 $O(\log n)$
- 即使输入是已排序的，随机化也能保证平均性能

**最坏情况**：仍然是 $O(n^2)$，但概率极低

### 在生物信息学中的应用

- **大规模数据排序**：比确定性快速排序更稳定
- **k-mer 计数**：某些工具使用随机化排序变种
- **分区统计**：快速找到中位数、分位数

## 2. Gibbs Sampling：从"抛硬币"到 Motif 发现

Gibbs Sampling 是一种马尔可夫链蒙特卡洛（MCMC）方法。在生物信息学中，它最经典的应用是解决 **Motif Finding** 问题。

### 为什么需要 Gibbs Sampling？
在 `Randomized Motif Search` 中，我们每次根据当前的 $t$ 个起始位置构建 Profile，然后寻找所有序列中最匹配的位置。这种方法虽然快，但非常容易陷入局部最优解（即找到了一个还可以但不是最好的模式）。

Gibbs Sampling 通过一种更"谨慎"的随机化方式来缓解这个问题：**它不是一次性更新所有序列的 Motif 位置，而是在每一步只随机选择一条序列进行更新。**

### 算法流程
1.  **初始化**：随机选择 $t$ 条序列的起始位置 $s_1, \dots, s_t$。
2.  **选择与移除**：随机选择一条序列 $i$（$1 \leq i \leq t$），暂时将其排除。
3.  **构建模型**：利用剩余的 $t-1$ 条序列中的 Motif 片段构建 Profile 矩阵 $P$。
4.  **概率估算**：在被移除的序列 $i$ 中，计算每一个可能的 $l$-mer 在模型 $P$ 下出现的概率。
5.  **按比例采样 (Roulette Wheel Selection)**：
    - 与贪心算法不同，我们不一定选概率最高的位置。
    - 我们按照计算出的概率分布随机采样一个新的位置 $s_i$。这意味着概率高的位置更可能被选中，但概率较低的位置也有机会，从而帮助算法跳出局部陷阱。
6.  **迭代**：重复上述过程 $N$ 次，记录并返回得分最高的 Motif。

### 关键直觉：抛硬币的平衡
Gibbs Sampling 就像是在一个多维度的"地形"中行走。如果只往高处走（贪心），可能会被困在一个小山丘上。Gibbs Sampling 允许你偶尔往"低处"走一步，从而有机会翻过障碍，到达更高的山峰。

详见：[Motif Discovery 算法路线](../models/motif-discovery-algorithms)

### 在生物信息学中的应用

- **Motif 发现**：寻找转录因子结合位点
- **序列比对**：某些比对算法使用采样策略
- **参数估计**：在复杂概率模型中估计参数

## 3. 随机投影 (Random Projections)

### 问题的背景
在 Motif Finding 问题中，如果我们想找一个长度为 $l$ 的模式，且允许最多 $d$ 个错配（称为 $(l, d)$-motif 问题），传统的枚举法或贪心法在 $l$ 较大时会变得极其缓慢。

### 随机投影的直觉
随机投影（Random Projections）是由 Buhler 和 Tompa 提出的一种巧妙的随机化方法。它的核心思路是：**如果我们无法处理完整的 $l$-mer，那么我们随机挑选其中的 $k$ 个位置进行"投影"。**

### 算法流程
1.  **选择模板**：随机选择 $k$ 个索引位置（$k < l$），例如 $\{1, 3, 5\}$。
2.  **投影与桶排序**：对于序列中的每一个 $l$-mer，只保留模板指定的 $k$ 个字符，将其投影为一个 $k$-mer。
3.  **收集候选者**：将投影后相同的 $k$-mer 放入同一个"桶"中。
4.  **识别潜在 Motif**：
    - 如果一个桶里的 $l$-mer 数量异常多，说明这些 $l$-mer 在投影位置上是一致的。
    - 它们很有可能就是我们要找的 $(l, d)$-motif 的变体。
5.  **验证与精修**：对高频桶中的 $l$-mer 进行进一步的共识序列计算和局部搜索。

### 为什么有效？
虽然单次投影可能会漏掉真正的 Motif（如果错配正好发生在我们挑选的 $k$ 个位置上），但由于算法运行非常快，我们可以通过**多次独立投影**来保证以极高的概率捕捉到真正的模式。

## 6. 复杂度分析

### 期望复杂度 vs 最坏复杂度

随机化算法通常分析**期望复杂度**（expected complexity）而非最坏复杂度。

**快速排序示例**：
- 最坏时间：$O(n^2)$
- 期望时间：$O(n \log n)$
- 实际中几乎总是接近期望时间

### 概率保证

某些算法提供概率保证：

- **成功概率**：算法以概率 p 返回正确答案
- **错误概率**：可以通过增加运行次数降低错误概率
- **高概率保证**：如 $1 - 1/n$ 的概率正确
