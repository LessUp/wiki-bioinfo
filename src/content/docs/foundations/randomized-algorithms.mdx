## 为什么需要随机化算法

确定性算法在处理某些问题时会遇到困难：

- **陷入局部最优**：贪心算法可能被困在局部最优解
- **算法复杂**：确定性算法可能需要复杂的分析
- **最坏情况差**：某些确定性算法的最坏情况性能很差

随机化算法通过引入随机性解决这些问题：

## 1. 快速排序的随机化

### 确定性快速排序的问题

确定性快速排序选择第一个元素作为 pivot：

- **最坏情况**：已排序数组，复杂度 $O(n^2)$
- **平均情况**：$O(n \log n)$
- **问题**：实际数据可能接近已排序，触发最坏情况

### 随机化快速排序

随机选择 pivot：

```
RANDOMIZEDQUICKSORT(array, low, high)
1 if low < high
2   pivot_index ← RANDOM(low, high)
3   swap array[low] and array[pivot_index]
4   pivot ← partition(array, low, high)
5   RANDOMIZEDQUICKSORT(array, low, pivot - 1)
6   RANDOMIZEDQUICKSORT(array, pivot + 1, high)
```

### 复杂度分析

**期望时间复杂度**：$O(n \log n)$

**关键观察**：
- 随机选择 pivot 使得每个元素有相等的概率成为 pivot
- 期望的递归深度是 $O(\log n)$
- 即使输入是已排序的，随机化也能保证平均性能

**最坏情况**：仍然是 $O(n^2)$，但概率极低

### 在生物信息学中的应用

- **大规模数据排序**：比确定性快速排序更稳定
- **k-mer 计数**：某些工具使用随机化排序变种
- **分区统计**：快速找到中位数、分位数

## 2. Gibbs Sampling

### 什么是 Gibbs Sampling

Gibbs Sampling 是一种马尔可夫链蒙特卡洛（MCMC）方法，用于从复杂的概率分布中采样。

### Motif 发现中的 Gibbs Sampling

#### 问题定义

在 t 条 DNA 序列中寻找一个长度为 l 的保守 motif。

#### 算法步骤

```
GIBBSSAMPLER(DNA, t, n, l, N)
1 随机选择每条序列的起始位置 s₁, s₂, ..., sₜ
2 for iteration ← 1 to N
3   随机选择一条序列 i（从序列中移除）
4   基于剩余 t-1 条序列构建位置权重矩阵（PWM）
5   根据 PWM 计算序列 i 中每个 l-mer 的概率
6   按概率采样新的起始位置 sᵢ
7   更新位置 sᵢ
8 返回最好的 motif
```

#### worked example

假设有 4 条序列，motif 长度为 6：

```
Seq1: ATGCGATGCGA...
Seq2: CGATCGATCGA...
Seq3: GATCGATCGAT...
Seq4: TCGATCGATCG...
```

1. 随机选择位置：[3, 5, 2, 4]
2. 随机选择移除 Seq1
3. 用 [5, 2, 4] 构建 PWM
4. 根据 PWM 计算 Seq1 各位置的概率
5. 按概率采样新位置
6. 重复 N 次，取最好结果

## 3. 随机投影（Random Projections）

### 基本思想

随机投影用于高维数据的降维和相似性搜索。

<DefinitionList
  items={[
    {
      term: '核心思想',
      definition: '通过随机映射将高维数据投影到低维空间，同时保持距离关系',
    },
    {
      term: 'Johnson-Lindenstrauss 引理',
      definition: '在足够高的低维空间中，点之间的距离得以保持',
    },
    {
      term: '应用',
      definition: '近似最近邻搜索、聚类、降维',
    },
  ]}
/>

### 在 Motif 发现中的应用

随机投影算法用于加速 motif 搜索：

#### 算法思想

1. 随机选择 k 个位置作为投影模板
2. 将所有 l-mer 投影到 k-mer 空间
3. 统计每个 k-mer 的出现次数
4. 选择高频 k-mer 对应的 l-mer 作为候选

#### worked example

假设 l=6, k=3，随机模板选择位置 \{1, 3, 5\}：

l-mer "ATGCGA" 投影为 k-mer "AGA"
l-mer "ATGCGT" 投影为 k-mer "AGT"

统计所有投影后的 k-mer 频率，高频 k-mer 对应的 l-mer 可能是 motif 的候选。

#### 优势

- **加速搜索**：将 4^l 的搜索空间降到 4^k
- **并行化**：可以同时使用多个随机模板
- **概率保证**：理论上可以证明找到正确 motif 的概率

### 在生物信息学中的应用

- **大规模序列搜索**：加速数据库检索
- **聚类分析**：高维基因表达数据的降维
- **特征选择**：从高维特征中选择重要特征

### Las Vegas vs Monte Carlo

| 特性 | Las Vegas | Monte Carlo |
|------|-----------|-------------|
| 正确性 | 总是正确 | 可能错误 |
| 运行时间 | 随机 | 确定 |
| 典型应用 | 随机化快速排序 | 随机投影、近似算法 |
| 概率保证 | 期望时间复杂度 | 成功概率 |

## 6. 复杂度分析

### 期望复杂度 vs 最坏复杂度

随机化算法通常分析**期望复杂度**（expected complexity）而非最坏复杂度。

**快速排序示例**：
- 最坏时间：$O(n^2)$
- 期望时间：$O(n \log n)$
- 实际中几乎总是接近期望时间

### 概率保证

某些算法提供概率保证：

- **成功概率**：算法以概率 p 返回正确答案
- **错误概率**：可以通过增加运行次数降低错误概率
- **高概率保证**：如 $1 - 1/n$ 的概率正确

## 常见误区

### 随机化算法不可靠

不是。随机化算法：
- 有严格的理论分析
- 提供概率保证
- 实际中往往比确定性算法更稳定

### 随机化总是比确定性慢

不一定。随机化算法：
- 平均性能通常更好
- 实现可能更简单
- 避免了最坏情况的触发

### 随机化只是运气

不是。随机化算法：
- 有精心的设计
- 随机性是工具，不是依赖运气
- 可以通过数学分析证明性能

### 只需要运行一次就够了

通常需要：
- 多次运行取最好结果
- 或者运行足够长时间确保收敛
- 或者使用集合方法（如随机森林）

## 与其他算法的连接

- **贪心算法**：随机化可以改进贪心，避免局部最优
- **动态规划**：某些问题可以用随机化近似求解
- **近似算法**：随机化是设计近似算法的重要工具
- **分治算法**：随机化快速排序是分治与随机化的结合
