本规范定义 BioInfo Wiki 的内容组织方式、术语标准和写作风格。所有贡献者应遵循这些约定，以保持知识库的一致性和可读性。

---

## 1. 页面类型与模板

根据内容性质，页面分为四种类型。每种类型有推荐的章节结构：

### 1.1 Concept page（概念页）

适合基础对象、数据库资源、数据格式、术语说明等页面：

```md
# 标题

## 是什么
## 为什么重要
## 核心概念
## 应用场景
## 常见误区
## 参考资料
## 相关页面
```

### 1.2 Algorithm page（算法页）

适合编辑距离、比对、组装图、HMM、索引结构等核心方法页面：

```md
# 标题

## 是什么
## 要解决什么生物信息学问题
## 输入与输出
## 核心思想 / 数学模型
## 关键递推 / 状态定义 / 图结构
## worked example
## 复杂度与适用前提
## 与真实工具或流程的连接
## 常见误区
## 参考资料
## 相关页面
```

### 1.3 Workflow page（流程页）

适合 RNA-seq、variant calling、metagenomics 等流程总览页面：

```md
# 标题

## 任务目标
## 输入输出
## 前置知识
## 步骤总览
## 每步依赖与常见错误
## 对应算法模块
## worked example
## 注意事项
## 后续阅读
## 相关页面
```

### 1.4 Section landing page（板块入口页）

适合一个大板块的入口页：

```md
# 板块标题

## 这一部分在全站中的位置
## 为什么这一节重要
## 学习前置
## 推荐阅读顺序
## 子主题导航
## 与其他板块的连接
```

## 2. 内容写作原则

建议尽量补齐下面这些元素：

- **定义**：先说清楚问题是什么；
- **动机**：为什么会需要这个算法、模型或流程；
- **形式化描述**：状态、递推、图模型、概率假设、输入输出；
- **例子**：至少有一个简短 worked example；
- **复杂度/限制**：说明方法在什么条件下成立；
- **交叉链接**：把前置知识、后续知识和同类方法连起来；
- **工具映射**：适当说明思想如何出现在真实软件或流程中。

## 3. 术语与语言规范

- 正文以中文为主；
- 关键术语首次出现时附英文，例如“隐马尔可夫模型（Hidden Markov Model, HMM）”；
- 数据库、软件、经典算法名称保留英文；
- 术语前后尽量保持一致，不要一页内混用多个译法；
- 如果一个名词可能有多种译法，请选一个主译法并在首次出现时说明。

## 4. 图表规范

- 每张图都应该服务于一个具体解释目标；
- 图片文件建议放在 `public/img/illustrations/` 下按专题组织；
- 图下注明图意，必要时说明数据来源或示意性质；
- 不要用装饰性图片代替解释；
- 能用一张流程图说明输入输出关系时，优先用图而不是长段文字。

### 4.1 公式规范

- 需要公式时优先用 LaTeX 数学块清晰写出递推关系或概率定义；
- 公式后应紧跟自然语言解释，不要只给符号不给含义；
- 表格适合用于比较比对策略、索引结构、不同文件格式、不同工作流选择；
- 如果一个对比会在多篇页面重复出现，优先使用可复用组件。

## 5. MDX 与组件使用

普通页面继续优先使用 Markdown；满足以下条件时再考虑 `.mdx`：

- 需要流程步骤卡片；
- 需要结构化对比表；
- 需要前置知识提示框；
- 需要一组可复用的相关页面卡片；
- 需要把图示和说明组织成更强的版式结构。

新增 MDX 页面时，优先复用站内组件，而不是每页自己拼样式。

## 6. 代码与命令

- 命令要能直接运行，或明确指出需要替换的部分；
- 示例输入输出尽量简短；
- 如果命令依赖上下文，先交代数据格式和目标；
- 对算法页来说，伪代码、矩阵示意和状态解释通常比冗长实现更重要；
- 对工作流页来说，命令只是辅助，重点仍然是为什么要做这一步。

## 7. 链接与引用

- 站内优先使用相对链接建立知识网络；
- landing page 要主动链接子页面，而不只是依赖 sidebar；
- 外链优先指向论文主页、官方文档或可信数据库；
- 引用教材和论文时尽量写明完整信息；
- 每个核心页面末尾都应有“相关页面”。

## 8. 内容层级

写新页面前，先判断它属于哪一层：

- **对象层**：reads、参考基因组、注释、文件格式；
- **模型层**：字符串、动态规划、图、概率模型；
- **算法层**：比对、组装、索引、motif、gene prediction；
- **流程层**：DNA-seq、RNA-seq、metagenomics 等工作流；
- **资源层**：数据库、参考版本、注释系统。

如果一篇页面同时跨了多层，建议主页面聚焦一层，其他层通过“相关页面”或“与真实工具连接”处理。

## 9. 页面发布检查清单

提交新页面或重大修改前，请确认以下事项：

### 内容质量
- [ ] 页面定位清晰，目标读者明确
- [ ] 包含问题背景或动机说明
- [ ] 核心概念、算法或流程已解释清楚
- [ ] 包含至少一个实例、图示或表格
- [ ] 术语使用一致，关键术语附英文

### 技术规范
- [ ] 文件格式为 `.md` 或 `.mdx`
- [ ] 使用了正确的 frontmatter（title 和 description）
- [ ] 站内链接使用相对路径
- [ ] 图片放入正确目录并有 alt 文本
- [ ] 运行 `npm run check` 无错误

### 知识网络
- [ ] 页面末尾包含"相关页面"章节
- [ ] 适当引用前置知识页面
- [ ] 如果创建了子页面，父页面已更新导航
