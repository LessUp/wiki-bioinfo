## 问题引入：为什么短读长难以检测 SV？

结构变异（SV）通常指长度 >50 bp 的基因组变异，包括：

| 类型 | 描述 | 示例场景 |
|------|------|----------|
| DEL | 缺失 | 肿瘤抑癌基因丢失 |
| INS | 插入 | 转座子插入、病毒整合 |
| DUP | 串联重复 | 基因剂量增加 |
| INV | 倒位 | 基因调控区域重排 |
| TRA | 易位 | BCR-ABL 融合基因 |

**短读长的局限**：

考虑一个 1 kb 的缺失变异：
- 短读长（150 bp）无法完整跨越缺失断点
- 需要依赖 split-read 或 read pair 间距异常等间接证据
- 在重复区域，这些间接信号极易产生假阳性

**长读长的优势**：
- 20 kb read 可直接包含整个缺失区域及其两侧序列
- 断点在单条 read 内可见，证据直接且明确
- 对复杂重排（如倒位-缺失组合）有更强解析能力

## 证据类型

### Split-Read：核心证据

**定义**：单条 read 的不同部分比对到参考基因组的不同（或相反方向）位置。

**生物学意义**：
- read 跨越了基因组重排断点
- 断点两侧的序列在同一 DNA 分子上相连

**示例**：
```
Read:    [ A段 ]---[ B段 ]
          ↓         ↓
Ref chr1: [ A段 ]------
Ref chr1:            [ B段 ]
         ↑断点位置↑
```

### Read Depth：辅助证据

- **深度降低**：可能提示缺失（DEL）
- **深度升高**：可能提示重复（DUP）
- **长读长局限**：覆盖度波动性大，单独使用不可靠

### Discordant Pattern：复杂重排

- **异常方向**：read 的两端比对到相反方向（提示倒位 INV）
- **跨染色体**：read 比对到不同染色体（提示易位 TRA）
- **大 gap**：read 内部有大段未比对区域（可能为插入 INS）

## 算法策略

### 检测流程

1. **比对**：Minimap2 将 reads 比对到参考
2. **证据提取**：识别 split-reads、异常比对模式
3. **聚类**：将支持同一 SV 的 reads 聚类
4. **断点精确定位**：整合多 read 信息确定断点边界
5. **基因型判定**：统计各等位基因的支持 reads 数

### 关键挑战

**重复区域**：
- 人类基因组 ~50% 为重复序列
- 重复序列内部的 SV 难以精确定位
- 需要重复注释辅助过滤

**复杂 SV**：
- 组合事件（如缺失+插入）
- 平衡重排（倒位、易位）无覆盖度信号
- 需要单倍型信息辅助解析

## 工具比较

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Sniffles2** | 速度快，支持 SV 分型 | 大规模队列研究 |
| **cuteSV** | 准确性高，支持复杂 SV | 高质量参考比较 |
| **SVIM** | 区分性组装方法 | 无参考或高度差异样本 |
| **pbsv** | PacBio 官方工具 | PacBio HiFi 数据 |

## 结果解释要点

SV 的生物学意义评估需考虑：

**基因影响**：
- 是否跨越基因编码区？
- 是否影响调控元件（启动子、增强子）？
- 阅读框是否保持？

**重复背景**：
- 是否位于已知重复区域？
- 是否为多态性而非致病变异？

**群体频率**：
- 在 gnomAD 等数据库中的频率？
- 是否为新发（de novo）变异？

## 常见误区

**单条 read 异常即判定 SV**
不可靠。需要多条 reads 支持，并排除比对错误和测序误差。

**断点精确到单碱基**
在重复区域或复杂重排中，断点往往只能定位到数百 bp 范围内。

**SV 大小决定重要性**
位置比大小更重要。一个 100 bp 的外显子缺失可能比 10 kb 的内含子缺失更有功能影响。

**所有平台检测结果一致**
不同平台（PacBio/Nanopore）和工具的结果可能有差异，建议多方法交叉验证。
