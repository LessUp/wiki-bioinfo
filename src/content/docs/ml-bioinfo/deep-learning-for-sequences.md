## 为什么序列也能做深度学习

DNA、RNA 和蛋白质本质上都可以视为符号序列，因此可以借鉴 NLP 和计算机视觉中的表示学习方法。不同之处在于：

- 字母表很小（DNA 只有 A/C/G/T）；
- 局部 motif、长程相互作用和结构约束都很重要；
- 标签常常稀缺且带噪声。

## 常见输入表示

### One-hot 编码

DNA 最经典的表示方式是把每个位点编码成长度为 4 的向量：

- A = `[1,0,0,0]`
- C = `[0,1,0,0]`
- G = `[0,0,1,0]`
- T = `[0,0,0,1]`

优点是简单直接，缺点是无法表达上下文语义。

### k-mer / embedding

也可以把序列切成 k-mer token，再映射为 embedding，和 NLP 中的词嵌入类似。

## 常见模型

### CNN

卷积网络非常适合识别局部 motif。很多早期 genomics 深度学习模型（如 DeepBind、DeepSEA）都使用 CNN：

- 卷积核可以学到 motif；
- 池化层汇总 motif 是否出现；
- 多层结构可进一步捕获组合模式。

### RNN

RNN/LSTM 曾用于建模长程依赖，但在长序列和大规模预训练上已逐渐被 Transformer 替代。

### Transformer

Transformer 的 self-attention 能直接建模远距离位置间的依赖，在蛋白质和基因组建模中都非常重要。典型优势：

- 更适合大规模预训练；
- 能提取上下文相关表示；
- 易于迁移到多种下游任务。

## 常见任务

- motif / binding site 预测；
- 启动子、增强子、剪接位点识别；
- variant effect prediction；
- 蛋白功能、亚细胞定位和结构属性预测；
- 单细胞扰动响应建模。

## 与经典方法的关系

深度学习并没有“替代一切”。在很多数据量较小或可解释性要求高的场景，PWM、HMM、线性模型依然非常有价值。更现实的理解是：

- 经典模型提供可解释的归纳偏置；
- 深度学习提供更强的表示能力；
- 两者常常在实际流程中结合使用。

## 常见误区

- **有了 Transformer 就不需要生物学知识**：输入设计、标签定义和评估方式仍高度依赖领域知识；
- **模型精度高就表示学到了机制**：预测性能和因果机制是两回事；
- **序列越长越好**：更长上下文并不总是带来更好结果，成本也更高。
