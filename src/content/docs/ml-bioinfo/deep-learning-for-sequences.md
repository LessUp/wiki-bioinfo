## 从符号序列到数值表示

生物序列（DNA、RNA、蛋白质）本质上是**离散符号的线性排列**。要让机器学习模型处理这些序列，首先需要解决一个基础问题：如何将符号序列转化为数值表示？

### 问题的核心

考虑一个 DNA 序列 `"ATGCGT"`。传统生物信息学方法会直接在这个字符串上操作（如字符串匹配、正则表达式）。但深度学习模型需要**连续数值输入**，因为：

1. 神经网络通过梯度下降学习参数，需要可微分的输入
2. 相似序列应该有相似的表示，便于泛化
3. 需要捕获上下文依赖和隐含模式

### 输入表示方法

#### One-hot 编码

最简单的方法是将每个碱基/氨基酸表示为**标准正交基向量**：

**DNA（4维）**：
| 碱基 | One-hot 向量 |
|------|-------------|
| A | `[1, 0, 0, 0]` |
| C | `[0, 1, 0, 0]` |
| G | `[0, 0, 1, 0]` |
| T | `[0, 0, 0, 1]` |

**蛋白质（20维）**：每个氨基酸对应一个 20 维向量。

长度为 $L$ 的序列被编码为矩阵 $X \in \mathbb{R}^{L \times K}$，其中 $K$ 是字母表大小。

**优点**：
- 简单直接，无信息损失
- 每个位置独立可解释

**缺点**：
- 无法表达碱基之间的相似性（如 A 和 G 都是嘌呤）
- 无上下文信息

#### k-mer 嵌入

借鉴 NLP 的**词嵌入**思想，将序列切分为 k-mer token：

```
序列: "ATGCGT"
3-mer: ["ATG", "TGC", "GCG", "CGT"]
```

每个 k-mer 通过学习获得一个稠密向量表示。这种方法：
- 捕获局部序列上下文
- 可学习的表示能表达 k-mer 间的相似性
- 计算效率较高（序列长度缩短为 $L-k+1$）

#### 化学性质编码

利用碱基或氨基酸的物理化学性质：

- **DNA**：分子量、氢键供体/受体数、芳香性
- **蛋白质**：疏水性、电荷、分子量、极性

这种方法将领域知识显式编码到输入中，适合某些特定任务。

## 三种核心架构的归纳偏置

不同的神经网络架构引入了不同的**归纳偏置**（Inductive Bias），即模型对数据结构的先验假设。选择架构就是选择适合特定生物问题的归纳偏置。

### 架构对比

| 架构 | 核心归纳偏置 | 优势场景 | 典型生物应用 |
|------|-------------|---------|-------------|
| **CNN** | 局部性、平移不变性 | motif 检测、局部模式识别 | 转录因子结合位点、调控元件 |
| **RNN** | 递归结构、时序依赖 | 序列生成、变长建模 | 基因预测、剪接位点 |
| **Transformer** | 全局注意力、长程依赖 | 复杂相互作用、大规模预训练 | 蛋白质结构、语言模型 |

### 为什么不同任务需要不同架构

**CNN 适合 motif 检测**的原因是：
- 转录因子结合的 motif 通常是局部且位置不变的
- 卷积核可以学习位置无关的 motif 检测器
- 池化层实现"是否存在"的判断，不关心具体位置

**RNN 适合基因预测**的原因是：
- 编码区的识别需要考虑阅读框和上下游信号
- 隐状态可以累积上下文信息
- 适合处理变长序列

**Transformer 适合蛋白质建模**的原因是：
- 氨基酸残基之间可能存在长程相互作用（如二硫键、疏水核心）
- 自注意力直接建模任意两个位置的关系
- 适合大规模预训练

## 典型任务类型

生物序列深度学习任务可按**输入-输出映射**分类：

### 序列级分类

**输入**：整条序列  
**输出**：单个标签或数值

- **调控元件识别**：判断序列是否为启动子/增强子
- **变异效应预测**：预测 SNP/Indel 对功能的影响
- **亚细胞定位**：预测蛋白质的细胞定位

### 位置级标注

**输入**：序列  
**输出**：每个位置的标签

- **基因预测**：识别编码区、UTR、内含子边界
- **剪接位点预测**：识别 5' 和 3' 剪接位点
- **翻译后修饰预测**：识别磷酸化、糖基化位点

### 序列到序列

**输入**：一条序列  
**输出**：另一条序列

- **序列生成**：生成具有特定性质的蛋白质序列
- **结构预测**：从序列预测二级结构或接触图
- **对齐**：序列比对、结构比对

### 对比与检索

**输入**：多条序列  
**输出**：相似度或排序

- **功能相似性检索**：找出功能相似的序列
- **聚类分析**：按序列特征对基因/蛋白分组

## 深度学习与经典方法的关系

深度学习并未完全取代 PWM、HMM 等经典方法。理解它们的互补关系很重要：

### 经典方法的优势

| 方法 | 优势 | 适用场景 |
|------|------|---------|
| **PWM/PSSM** | 可解释性强，参数少 | 已知 motif 的检测，小样本 |
| **HMM** | 概率框架完整，有理论基础 | 基因预测，需要考虑状态转移 |
| **k-mer 计数** | 简单高效，无需训练 | 序列分类，快速筛选 |

### 深度学习的优势

- **表示学习**：自动发现重要特征，无需手工设计
- **非线性建模**：捕获复杂的特征交互
- **迁移学习**：预训练表示可迁移到多个下游任务

### 混合策略

实际应用中常常结合使用：

1. **CNN + PWM**：用 CNN 提取特征，用 PWM 约束卷积核初始化
2. **深度学习 + HMM**：深度学习提供发射概率，HMM 提供状态转移约束
3. **预训练嵌入 + 经典分类器**：用语言模型提取 embedding，再接 SVM/随机森林

## 模型选择与评估原则

### 如何选择架构

1. **数据量**：小样本优先考虑经典方法或简单 CNN；大数据可尝试 Transformer
2. **可解释性需求**：需要理解机制时优先 CNN（可可视化卷积核）或经典方法
3. **长程依赖程度**：局部 motif 用 CNN；长程相互作用用 Transformer
4. **计算资源**：Transformer 需要更多算力，RNN 推理较慢

### 常见评估陷阱

| 陷阱 | 说明 | 避免方法 |
|------|------|---------|
| **数据泄漏** | 测试序列与训练序列高度相似 | 使用序列 identity 阈值划分数据集 |
| **不平衡数据** | 正负样本比例悬殊 | 使用 stratified sampling，关注 F1/AUROC |
| **过拟合** | 训练集表现好，测试集差 | 早停、交叉验证、正则化 |
| **评价指标不当** | 高准确率可能无意义 | 根据任务选择 AUROC、AUPRC、MCC 等 |

### 重要提醒

> **预测性能 ≠ 机制理解**
>
> 模型在高准确率并不等同于它"理解"了生物机制。一个 CNN 可能通过统计相关性做出准确预测，但这与揭示因果机制是两回事。解释性分析（如 saliency maps、attention visualization）是必需的后续步骤。
