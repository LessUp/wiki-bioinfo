## 从监督学习到自监督预训练

传统生物序列分析通常遵循**监督学习**范式：收集标注数据 → 训练模型 → 预测新数据。但生物数据面临一个根本挑战：

> **标注数据稀缺，无标注数据海量**

基因组数据库中有数十亿条序列，但带有功能注释的只有一小部分。语言模型的核心思想是**自监督预训练**：从海量无标注序列中学习通用表示，再迁移到下游任务。

### 自监督学习的原理

**核心思想**：设计不需要人工标注的预训练任务，让模型从序列本身的结构中"自动生成"学习信号。

**典型预训练任务**（以蛋白质为例）：

| 任务 | 做法 | 学习目标 |
|------|------|---------|
| **掩码语言建模（MLM）** | 随机掩盖部分氨基酸，预测被掩盖的是什么 | 理解氨基酸之间的共现关系 |
| **自回归建模** | 从左到右逐个预测下一个氨基酸 | 学习序列的生成概率 |
| **对比学习** | 对同一条序列做不同扰动，拉近其表示 | 学习鲁棒的序列表示 |

### 为什么自监督有效

1. **数据效率**：预训练利用全部可用序列，微调只需少量标注
2. **通用表示**：预训练学到的特征适用于多种下游任务
3. **隐式知识**：模型从海量序列中"学习"了进化约束和功能模式

## 预训练-微调范式

生物语言模型通常遵循两阶段流程：

### 阶段一：预训练（Pre-training）

**输入**：大规模未标注序列（如 UniProt 的数百万蛋白序列）  
**目标**：通过自监督任务学习通用表示  
**输出**：预训练模型权重

这一阶段计算成本高（需要 GPU/TPU 集群训练数天到数周），但只需执行一次。

### 阶段二：微调（Fine-tuning）

**输入**：特定任务的标注数据  
**方法**：
- **冻结表示**：用预训练模型提取 embedding，训练简单的下游分类器
- **全模型微调**：在特定任务上继续训练整个模型，调整所有参数
- **Adapter/Lora**：只训练少量新增参数，保持预训练权重不变

**输出**：适配特定任务的模型

### 迁移学习的优势

| 对比项 | 从头训练 | 预训练+微调 |
|-------|---------|-------------|
| 所需标注数据 | 大量 | 少量即可 |
| 训练时间 | 长 | 短（微调阶段） |
| 泛化能力 | 依赖数据分布 | 通常更好 |
| 计算成本 | 高 | 预训练一次性，微调低成本 |

## 代表性模型

### 蛋白质语言模型

#### ESM（Evolutionary Scale Modeling）

**核心特点**：
- 基于 Transformer 架构
- 使用掩码语言建模预训练
- 模型规模从 ESM-1b（650M 参数）到 ESM-2（15B 参数）

**下游应用**：
- **突变效应预测**：预测点突变对蛋白功能的影响
- **结构预测**：从序列预测接触图、二级结构
- **功能注释**：零样本功能预测

#### ProtTrans

**核心特点**：
- 基于自回归模型（类似 GPT）
- 在 BFD 和 UniRef 上预训练
- 提供多种规模的模型（ProtT5、ProtBert）

#### AlphaFold 的表示

AlphaFold2 中的 Evoformer 可以看作是结构感知的序列表示学习器，其输出的表示编码了：
- 共进化信息（MSA 特征）
- 结构约束（配对表示）
- 物理化学约束

### DNA/RNA 语言模型

#### DNABERT

**核心特点**：
- 基于 BERT 架构的 DNA 语言模型
- 使用 k-mer tokenization（k=3,4,5,6）
- 在人类参考基因组上预训练

**应用场景**：
- 启动子、增强子识别
- 染色质状态预测
- 变异效应预测

#### Nucleotide Transformers

**核心特点**：
- 使用字节对编码（BPE）进行 tokenization
- 在多种物种基因组上预训练
- 针对长序列优化

### 多物种与进化感知模型

一些模型显式利用进化信息：

- **MSA Transformer**：在多重序列比对（MSA）而非单序列上训练
- **Evoformer**（AlphaFold）：结合序列和配对表示
- **ESM-MSA**：利用 MSA 中的共进化信号

## 表示的层次与使用

### 多层次表示

语言模型可以提取不同层次的表示：

#### 1. Token-level Embedding（残基层/碱基层）

每个位置一个向量，维度通常为 768-1280。

**用途**：
- 位置特异性预测（如修饰位点、结构特征）
- 注意力可视化（观察哪些位置相互作用）

#### 2. Sequence-level Embedding（序列层）

整条序列的汇总表示，通过池化（mean/max）或特殊 token 获得。

**用途**：
- 序列分类（功能、定位）
- 相似性搜索
- 聚类分析

#### 3. Attention Map（关系层）

Transformer 的自注意力权重反映了位置之间的关联强度。

**用途**：
- 识别长程相互作用
- 可视化模型"关注"哪些位置
- 辅助解释预测结果

### 下游使用方式

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   原始序列       │ ──▶ │   语言模型       │ ──▶ │   Embedding     │
│  (ATGC.../MVK...)│     │  (ESM/DNABERT)  │     │  (向量表示)      │
└─────────────────┘     └─────────────────┘     └─────────────────┘
                                                          │
        ┌─────────────────┬─────────────────┬───────────────┴───────────┐
        ▼                 ▼                 ▼                           ▼
┌───────────────┐ ┌───────────────┐ ┌───────────────┐     ┌───────────────┐
│ 线性分类器     │ │   相似性搜索   │ │  聚类分析     │     │  多模态整合   │
│ (SVM/MLP)     │ │   (k-NN/ANN)  │ │  (K-means)    │     │  (+表达数据)  │
└───────────────┘ └───────────────┘ └───────────────┘     └───────────────┘
```

## 模型选择与限制

### 如何选择预训练模型

| 考量因素 | 建议 |
|---------|------|
| **数据类型** | 蛋白选 ESM/ProtTrans；DNA 选 DNABERT/Nucleotide Transformer |
| **序列长度** | 短序列（<1000）可用标准模型；长序列需特殊架构（HyenaDNA） |
| **下游任务** | 结构预测考虑 ESMFold；功能预测考虑 ESM-2；调控分析考虑 DNABERT |
| **计算资源** | 资源有限时选用小模型或冻结表示 + 轻量分类器 |

### 使用限制

| 限制 | 说明 | 应对策略 |
|------|------|---------|
| **计算成本** | 大模型推理慢、显存占用高 | 使用小模型、模型量化、批处理 |
| **可解释性弱** | 嵌入不像 PWM 那样直观 | 注意力可视化、 probing 分析 |
| **领域偏移** | 预训练数据与目标任务分布可能不同 | 领域适应、持续预训练 |
| **数据泄漏** | 预训练数据可能包含测试序列 | 严格的序列 identity 过滤 |

## 常见误区

### "Embedding 相似 = 功能相同"

**误区**：认为在高维空间中接近的序列一定功能相同。  
**事实**：Embedding 相似只说明统计模式相似，不等于功能相同。进化关系、结构约束都会影响表示。

### "预训练模型可以替代实验"

**误区**：认为语言模型预测足够准确，可以不做实验验证。  
**事实**：它们是高质量的先验和排序工具，但实验验证仍是必要的。预测应指导实验设计，而非替代实验。

### "越大越好"

**误区**：认为模型参数越多越好。  
**事实**：模型规模需要与数据量和任务复杂度匹配。对于小样本任务，过大模型容易过拟合。

### "零样本预测足够可靠"

**误区**：直接使用预训练模型做零样本预测，不做任何微调。  
**事实**：零样本预测可作为基线，但微调通常能显著提升性能。
