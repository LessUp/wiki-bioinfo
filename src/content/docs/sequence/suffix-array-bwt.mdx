## 1. Suffix Array (后缀数组)

**定义**：对于长度为 $n$ 的文本 $T$，其 Suffix Array $SA[0 \dots n-1]$ 是一个整数数组，记录了 $T$ 的所有后缀按字典序排序后的起始位置。

### 示例：$T = \text{BANANA\$}$
| 索引 | 后缀 | $SA$ 值 | 字典序排名 |
| :--- | :--- | :--- | :--- |
| 6 | `$` | 6 | 0 |
| 5 | `A$` | 5 | 1 |
| 3 | `ANA$` | 3 | 2 |
| 1 | `ANANA$` | 1 | 3 |
| 0 | `BANANA$` | 0 | 4 |
| 4 | `NA$` | 4 | 5 |
| 2 | `NANA$` | 2 | 6 |

- **查询**：通过对 $SA$ 进行**二分查找**，可以在 $O(m \log n)$ 时间内找到长度为 $m$ 的模式串。

## 2. Burrows-Wheeler Transform (BWT)

BWT 并不是一种简单的压缩算法，而是一种**排列变换**。它将文本中具有相似上下文的字符聚拢在一起。

### 构造直觉
1.  列出文本 $T$ 的所有循环位移（Rotations）。
2.  将这些位移按字典序排序形成一个矩阵。
3.  **BWT 串**即为该矩阵的最后一列。

**关键性质**：BWT 串中的第 $i$ 个字符正好是 Suffix Array 中排名第 $i$ 的后缀在原文本中的前一个字符。

## 3. 后向搜索 (Backward Search)

这是 FM-index 的核心黑科技：**从后往前匹配模式串**。

### 算法逻辑
假设我们要查 `ANA`：
1.  **查找 `A`**：在排序后的第一列（First Column）中找到所有以 `A` 开头的后缀区间。
2.  **查找 `NA`**：利用 BWT 串（Last Column）和 **LF 映射 (Last-to-First mapping)**，找到那些后面跟着 `A` 且自身是 `N` 的位置。
3.  **查找 `ANA`**：重复上述过程。

**LF 映射的直觉**：BWT 矩阵最后一列中的第 $k$ 个字符 `X`，对应于第一列中的第 $k$ 个字符 `X`。这允许我们在不存储完整矩阵的情况下，在索引空间内自如穿梭。

## 4. FM-index：现代工业标准

FM-index (Full-text index in Minute space) 将 BWT 与两个关键的辅助结构结合：
- **C[c]**：统计在字典序中比字符 $c$ 小的所有字符总数。
- **Occ(c, i)**：记录字符 $c$ 在 BWT 串的前 $i$ 个位置中出现了多少次。

**优势**：
- **空间极省**：其大小通常接近文本的压缩熵，人类基因组索引仅需约 2-4 GB。
- **查询极快**：匹配时间仅与模式串长度 $m$ 相关，与文本长度 $n$ 无关。
