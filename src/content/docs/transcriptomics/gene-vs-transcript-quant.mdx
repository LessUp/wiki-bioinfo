## 1. 定量的“解析度”之争

### 转录本层 (Transcript-level)
直接估计每个具体 Isoform 的表达量。
- **优点**：能捕捉选择性剪接（Alternative Splicing）和转录本切换现象。
- **挑战**：由于不同 Isoform 共享大量外显子，Reads 往往具有**多重比对 (Multi-mapping)** 性质，导致估计结果具有较高的方差。

### 基因层 (Gene-level)
将属于同一个基因的所有转录本信号合并处理。
- **优点**：抵消了 Isoform 间分配的不确定性，统计灵敏度（Sensitivity）更高，且与 GO/KEGG 等基因功能数据库直接兼容。
- **缺点**：掩盖了复杂的剪接变化（如基因总表达量不变但 Isoform 比例发生了变化）。

## 2. 聚合算法 (Aggregation)

如果已经获得了转录本层的定量值，可以通过以下方式转化为基因层：

### 简单求和 (Sum Aggregation)
对于 TPM 指标，由于已经校正过长度，可以直接求和：
$$\text{TPM}_{Gene} = \sum \text{TPM}_{Isoform}$$

### 长度校正聚合 (Length-corrected Aggregation)
对于原始计数 (Counts)，不能简单相加（因为 Isoform 长度不同）。常用工具（如 `tximport`）采用：
$$\text{Counts}_{Gene} = \sum \text{Counts}_{Isoform} \cdot \frac{\text{MeanLength}_{Gene}}{\text{Length}_{Isoform}}$$
这保证了聚合后的基因计数仍能反映出原本的丰度比例。

## 3. 分析策略指导

| 场景 | 推荐层级 | 原因 |
| :--- | :--- | :--- |
| **标准差异表达分析** | **基因层** | 统计模型（DESeq2）更稳健，假阳性低。 |
| **寻找新生物标志物** | **转录本层** | 有些癌症特异性标志物仅在特定 Isoform 上体现。 |
| **功能通路分析** | **基因层** | 绝大多数数据库是以 Gene ID 为核心构建的。 |
| **剪接调控研究** | **转录本层** | 必须区分不同的剪接变体。 |

## 4. 常见误区

- **先做基因层比对再定量**：直接把 Reads 比对到基因区间会由于忽略了剪接结构而导致严重的错误。正确的流程通常是：**比对到转录组/基因组 $\to$ 转录本层定量 $\to$ 聚合到基因层**。
- **忽略 Isoform Switching**：如果一个基因的 Isoform A 减少而 Isoform B 增加，基因层的总表达量可能显示“无显著变化”，从而漏掉重要的生物学发现。
