## 是什么

Pseudo-alignment / lightweight mapping 的核心目标不是为每个 read 找到精确的碱基级比对路径，而是更快地回答：

- 这个 read 与哪些转录本兼容；
- 这些兼容关系如何支持表达定量。

换句话说，它把问题从“精确对齐”转向“候选转录本归属与丰度估计”。

## 为什么重要

RNA-seq 中一个关键困难是：

- reads 很多；
- 转录本高度相似；
- 同一个 read 往往能同时匹配多个 isoform。

如果对每个 read 都做昂贵的碱基级比对，计算成本会很高；而如果只是为了做表达定量，我们更关心的是它支持哪些转录本，而不是每个碱基如何一一对齐。

## 核心概念

### 兼容性而不是精确路径

Pseudo-alignment 通常依赖：

- k-mer 或类似局部片段建立索引；
- 读入一个 read 后，快速找到与之兼容的一组转录本；
- 再通过统计模型把 reads 的证据分配到这些转录本上。

它强调的是**兼容性**，而不是完整的碱基级路径恢复。

### 与传统 spliced alignment 的区别

| 维度 | 传统比对 | Pseudo-alignment |
|---|---|---|
| 主要目标 | 精确定位 read 与参考之间的对齐关系 | 快速判断 read 与哪些转录本兼容 |
| 适合场景 | 剪接分析、定位解释、新转录本研究 | 表达定量与高通量样本处理 |
| 计算代价 | 通常更高 | 通常更低 |

### 多重归属与丰度估计

Pseudo-alignment 并不回避“一个 read 兼容多个转录本”这个现实，相反，它正是围绕这一点设计的：

- 先记录兼容集合；
- 再用整体 reads 分布估计各转录本丰度。

## worked example

假设一个 read 可以同时匹配两个共享外显子的转录本：

- `tx1`
- `tx2`

Pseudo-alignment 不一定为它恢复一条精确的碱基级比对路径，而是记录：

```text
read_42 -> {tx1, tx2}
```

后续定量模型再结合更多 reads 的整体分布，判断 `tx1` 和 `tx2` 各自更可能有多少丰度。

## 与真实工具或流程的连接

这类方法常用于：

- 快速转录本层表达定量；
- 大样本批量处理；
- 把计算重点放在丰度估计而不是定位细节上。

如果研究重点是：

- 精确剪接边界；
- 新结构发现；
- 局部变异解释；

则通常仍需要回到更精细的 spliced alignment 框架。

## 常见误区

- pseudo-alignment 等于不需要索引或算法；
- 既然更快，就应该总是取代传统比对；
- read 落到多个转录本上说明结果不可靠。
