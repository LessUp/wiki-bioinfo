## 是什么

DNA-seq 变异过滤（variant filtering）是在 caller 产生一批候选变异之后，对这些候选根据证据强度和上下文进行筛选的过程。

它的目标不是“找到所有差异”，而是：

- 尽量保留可信的生物学变异；
- 尽量剔除由测序错误、比对错误或模型局限引起的假阳性；
- 在召回率和精确率之间做合理平衡。

## 为什么重要

真正困难的往往不是 caller 跑出结果，而是判断哪些位点值得信。过滤这一层连接了：

- BAM 中的原始 read 证据；
- VCF 中的质量字段；
- caller 自带的过滤逻辑；
- 重复区域、低复杂度区域和样本背景。

如果过滤策略不合理，后续解释、数据库查询和统计分析都会建立在不稳定的候选集上。

### Hard filter 与模型化过滤

**Hard filter**：直接用阈值裁剪，例如深度、质量或 strand bias 低于阈值就剔除。

优点是直观；缺点是很难对不同平台和实验条件自适应。

**模型化过滤**：使用多个字段联合评估可信度。

优点是能结合更多证据；缺点是更黑箱，跨项目迁移时要重新理解假设。

### 区域上下文

即使字段看起来不错，在重复序列和低复杂度区域仍可能出现：

- 多重比对导致定位不稳定；
- mapping 错位导致假阳性 indel 或 SNP；
- coverage 异常堆积掩盖真实模式。

## worked example

在教学或探索阶段，可以先用一个较为保守的思路：

- 看 `FILTER = PASS`；
- 看深度是否过低；
- 看等位基因支持是否只来自极少数低质量 reads；
- 对重复和低复杂度区域额外谨慎。

需要强调的是：**不存在一组对所有数据都正确的通用阈值。**

过滤策略必须结合：

- 测序平台与实验设计；
- caller 的具体字段定义；
- 下游任务对假阳性和假阴性的容忍度。

## 与真实工具或流程的连接

真正的 variant filtering 不是看单个字段，而是把以下证据一起看：

- BAM 中的 read 支持；
- VCF 中的质量和偏好性指标；
- caller 文档中的 FILTER 含义；
- 区域注释、群体频率与功能注释。

因此，它是连接 caller 输出与结果解释的一层关键桥梁。

## 常见误区

- 仅凭单个字段（例如 QUAL）做一刀切过滤；
- 把某套阈值原封不动复制到不同平台和 caller；
- 忽略重复或低复杂度区域的特殊性；
- 把 PASS 理解成“绝对真实变异”。
